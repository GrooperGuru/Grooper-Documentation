Scriptable Types

Grooper provides a number of Objects that support Scripting. Each of these Objects is considered a different Scriptable Type and is used to accomplish specific scripting tasks, as follows:  

Scriptable Type Description
Batch Process Script Batch Processing scripts are event-driven, and work by responding to events raised by the Batch Process object. Batch Processing scripts are typically used to perform customizations such as:
  • Modifying the flow of the Batch Process.
Data Model Script Data Model scripts are event-driven, and work by responding to events raised by the Data Model object. Data Model scripts are typically used to perform customizations such as:
  • Complex data validation which cannot be implemented using the Validation Expression and Calculate Expression properties of the Data Field object.
  • Complex database lookups which cannot be implemented using the built-in lookup mechanisms of the Data Field object.
  • Add custom toolbar buttons to the Index Panel.
  • Perform custom processing and manipulation of index data.
Data Table Script Data Table scripts are event-driven, and work by responding to events raised by the Data Table object. Data Table scripts are typically used to perform customizations such as:
  • Perform custom data validation on the Data Table.
Data Type Script Data Type scripts are event-driven, and work by responding to events raised by the Data Type object. Data Type scripts are typically used to perform customizations such as:
  • Perform custom data validation on the Data Type.
Object Library Script Object Library scripts can be used to perform custom programming using many of the objects available in Grooper. Object Library scripts are typically used to perform customizations such as:
  • Creating a custom Activity inherited from AttendedActivity or UnattendedActivity (Unattended Activity Example) that can be added as a Batch Process Step in any Batch Process.
  • Creating a custom Windows Service inherited from TimerServiceInstance. (Timer Service Example)
  • Creating a custom Web Service inherited from WebService. (Web Service Example)
  • Creating a custom Report Definition inherited from ReportDefinition or SimpleReportDefinition.
  • Creating a custom Capture Device inherited from CaptureDevice.
  • Creating a custom IP Command inherited from IPCommand. (IP Command Example)
  • Creating a custom OCR Engine inherited from OCREngine.
  • Creating a custom Zonal Capture inherited from CaptureZone.
  • Creating custom Property Grids that are used as the configuration UI in other Object Library scripts. (Property Grid Editor Examples)
  • Creating a custom Object Command that is used from the context menus of Batch Pages and Batch Folders from the Batch Viewer. (Object Command Example)
