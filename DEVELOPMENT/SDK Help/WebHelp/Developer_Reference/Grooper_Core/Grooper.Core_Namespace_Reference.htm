Grooper.Core Namespace Reference

Classes

Class Name Description
AbsoluteAdjustment Adjusts the edge to a specific absolute position.
AnchorAdjustment Adjusts the edge relative to a text element on the document.
ArrayProvider Matches a list of values arranged in horizontal, vertical, or flow order.
AttendedActivity Serves as the base class for all attended activities.
BaseArrayProvider
BaseCombiningProvider
Batch A Batch is a hierarchy of folders and pages which serves as the fundamental unit of capture in Grooper.
BatchesFolder A Batches Folder object is a folder which contains Batches.
BatchFolder The Batch Folder object represents a logical folder within the structure of a Batch.
BatchObject Batch Objects represent items that can appear in Batches. Currently these are Batch Folders and Batch Pages.
BatchObjectStreamGenerator Represents a base class for objects which generate an output stream using the contents of a BatchFolder object.
BatchPage The Batch Page object represents a logical page within a Batch.
BatchPage: ImageInfo
BatchProcess A Batch Process defines a list of steps through which batches will be processed.
BatchProcessFolder Represents the 'Batch Processing / Processes' folder found in the root of a Grooper repository.
BatchProcessingActivity Activities define work to be performed on objects in a batch, and define settings which control how the work is performed.
BatchProcessingFolder Represents the 'Batch Processing' folder found in the root of a Grooper repository.
BatchProcessStep Represents a logical step in a Batch Process.
CaptureZone Capture Zones associate regions on a page with Data Elements in a Data Model, and are used to facilitate zonal data capture.
ClassifyMethod Classify Metheds define classification mechanisms which are used in Content Models.
CollationProvider Collation providers define various methods of processing the the output of a Data Type object.
CombineProvider Combines instances from child extractors based on the grouping specified in the Group By property.
ContainerDataElement A Data Element that is a container for one or more other data elements in a data element hierarchy.
ContentCategory A Content Category is a container for other Content Category and Document Type objects within a Content Model.
ContentLink Content Links define references to content stored outside of Grooper, such as file on a file system or a document in a CMIS Repository.
ContentModel A Content Model defines the taxonomy of a document set, in terms of the Document Types it contains, and the Data Elements which appear on each document type.
ContentType A Content Type is any element of a Content Model.
DatabaseTable A Database Table object represents an externally-stored database table which can be queried from Grooper.
DataColumn Represents a column in a Data Table.
DataConnection A Data Connection object represents a connection to a database, storing all information necessary to access the database. These objects provide a connection basis for Database Table objects, and are also used directly in scripts to avoid storing authentication credentials in script code.
DataElement Represents an element of data.
DataElementInstance Defines a DataInstance which represents the value of a Data Element in a Data Model.
DataElementProfile Data Element Profiles store settings for Data Elements such as sections, tables, and fields on a Content Type.
DataExtractionFolder Represents the 'Data Extraction' folder found in the root of a Grooper repository.
DataField Represents a field or table cell.
DataFormat Data Format objects are children of Data Type objects, and represent a distinct format for the Data Type. For example, a date could appear on one document as 05/17/2013, and as May 17, 2013 on another document. Each of these formats is defined by a Data Format object which is a child of the 'Date' Data Type object.
DataInstance Defines an instance of text data on a document.
DataModel Defines the data elements for a content type.
DataPattern Defines an extractor which returns all instances of data matching a regular expression.
DataPattern: GroupLookupOptions Defines lookup options which apply to a named regular expression group.
DataPattern: LookupOptions Defines lookup options which should be applied to extraction results.
DataPattern: TextPreprocessor Defines options for temporary text preprocessing which occurs prior to executing an extractor.
DataPattern: TextPreprocessor+HorizontalTabMarker Defines settings which control horizontal tab insertion.
DataPattern: TextPreprocessor+ParagraphMarker Defines settings which control paragraph detection.
DataPattern: TextPreprocessor+VerticalTabMarker Replaces \r\n pairs with \v characters in cases where the vertical gap between two lines of text meets certain criteria.
DataSection A Data Section allows the content of a document to be subdivided into sections for further processing. Sections can be used for a variety of purposes, including but not limited to the following:
  • To identify and extract repeating sections of a document.
  • To break a document down into a hierarchy of sections and subsections, representing the structure of the document's content.
  • To reorder text flow in situations where a document has multiple columns of information.
  • To simplify data extraction on complex documents.
DataTable Data Table objects are used to describe tabular data for the purposes of Data Extraction and Data Review. The columns in the table are defined by adding child Data Field elements to a Data Table object.
DataType A Data Type defines extraction logic for a distinct type of data, such as a field value or a table row. Each data type defines one or more extractors, along with settings which control how the extractor results are transformed into a final result set.
DividerSEM Identifies Section Instances by splitting the input content at each occurence of a specific text pattern.
DocumentInstance Represents the entire content of a document, and serves as the root of the DataInstance hierarchies generated by the Extraction activity.
DocumentType A Document Type is an object in a Content Model which represents a distinct type of document, such as an invoice or contract. Document Types are ultimately assigned to documents in a batch in a process called classification
EdgeAdjustment Defines a mechanism for adjusting edges of a region in Geometric section extraction.
EdgeOfPageAdjustment Adjusts the edge to the full bounds of the page.
ElementContainerInstance Represents a data element instance which may contain child sections and tables.
EmbeddedExtractor Represents a property which can (a) define a local extractor in the form of a Data Pattern object; or (b) or reference an external extractor such as a Data Type or Field Class.
EmbeddedLexicon Represents a property which can include locally-defined lexicon entries as well as references to external lexicons.
ExportProvider Export providers define export connectivity to external systems such as file systems, content management repositories, mail servers, and etc.
ExtractZone Serves as the base class for all extraction zones.
FieldClass A Field Class is a trainable binary classifier, used for locating specific instances of a data pattern on a document.
FieldClassInstance Represent a DataInstance captured by a Field Class.
FieldContainerInstance Defines an instance which may contain field objects as descendants.
FieldInstance Represents the value of a Data Field object.
FieldProfile Defines settings for a Data Field which apply when processing a specific Content Type.
FieldZone Field Zones associate a Data Field with a region of the document, and optionally specify zonal extraction parameters.
FileExport Provides import connectivity to CMIS-compliant content management platforms.
FileImport Provides common functionality to all file system-based import providers.
FileSystemExport Provides export connectivity to the Windows file system.
FileSystemImport Provides import connectivity to file systems.
FileSystemLink Defines a link to a file stored on the Windows file system.
FixedSEM Identifies section instances using a fixed geometric region.
FlowLayoutProvider Detects cases where the array elements occur one after another in the text flow of the document.
FlowSEM Identifies Section Instances using an extractor to match segments of text. For each match, the character data between the starting and ending position will become an output Section Instance.
FolderContentGenerator Defines settings which control the export of content from a Batch Folder.
FormType A Form Type represents a variation or version of a Document Type.
GeometricProvider Defines a rectangular region using anchors, and extracts the character data bounded by the region as a Section Instance.
GrooperVoting A Grooper OCR which performs OCR using multiple OCR Profiles, and then combines the results using a voting mechanism.
HeaderValue Detects the layout of a table automatically by analyzing results from value and header extractors defined on Data Column objects.
HorizontalLayoutProvider Detects cases where the array elements are arranged in a horizontal row.
HvHeaderDetector The header detection process attempts to detect a table header on each page, by analyzing results from the header extractors defined on individual Data Column objects.
HvRowDetector The row detection process uses the Value Extractors defined on primary columns to detect individual table rows.
ImportProvider Import providers enable import of file-based content from a variety of sources such as file systems, mail servers, and content repositories.
ImportProvider: FolderLevelInfo Defines folder levels which will be translated into subfolders within the Grooper Batch.
InferGrid Extracts tables which have both row and column headers, by infering a grid from the header positions.
InstanceFilter Defines rules for excluding individual results from the output.
LayoutProvider Layout Providers define the various layouts in which arrays can be detected.
Lexical Classifies documents or pages based on the their text content.
Lexicon A Lexicon is a dictionary which stores a list of keys or key-value pairs which are indexed for high-speed lookup.
MimeTypeHandler MIME type handlers define properties and commands which may be applied to a Batch Folder object with a specific MIME type.
NormalProvider Returns all instances from all extractors individually.
OmrZone OMR Zones are used for Optical Mark Recognition (OMR), to read the state of checkboxes and determine if they are checked or unchecked.
OrderedArrayProvider Finds sequences of values where one result is present for each extractor, in the order in which they appear.
PageType A Page Type represents a logical page within a Form Type. A Form Type with 5 pages will contain 5 Page Types as children. Page Types are not configured manually by users, but created internally by Grooper as documents are trained into a Content Model, and are used to store sample pages and the features extracted from those sample pages. The Page Type object has no configurable properties.
PatternBasedProvider Uses a regular expression to select a sequence of child extractor results.
PdfMimeTypeHandler MIME type handler for PDF document types. Defines the PDF-specific properties and commands which may be appied to a Batch Folder object.
ProfilesFolder Represents the 'Global Resources' folder found in the root of a Grooper repository.
Project A Project is defined as a collection of billing items, and can be assigned to Batch Processes.
Project: BillingItem A Billing Item consists of an Activity, a Statistic Name and a Rate.
RegistrationZone Registration zones are used to align Field Zones and OMR Zones prior to zonal extraction.
RowMatch Uses an extractor to match each row.
SectionExtractMethod A Section Extract Method defines an extraction method for use with Data Section objects.
SectionInstance Represents the value of a Data Section object.
SectionProfile Defines settings for a Data Section which apply when processing a specific Content Type.
SingleInstanceProvider Returns the first instance returned by any extractor and discards all other results. Useful for implementing a waterfall approach where the most strict format is tried first, follow by less strict formats.
SplitProvider Splits the input at each instance found by an extractor. The Split Position property provides a variety of methods for splitting the input.
StorageType
StorageTypeBoolean Represents a Boolean (true or false) value.
StorageTypeDateTime Represents an instant in time, typically expressed as a date and/or time of day.
StorageTypeDecimal Represents a decimal value.
StorageTypeDouble Represents a 64-bit floating point value.
StorageTypeGUID Represents a globally unique identifier (GUID).
StorageTypeInt16 Represents a 16-bit integer value.
StorageTypeInt32 Represents a 32-bit integer value.
StorageTypeInt64 Represents a 64-bit integer value.
StorageTypeNumeric Base class for all numeric types.
StorageTypeRanged Base class for all integer storage types.
StorageTypeString String values can store any type of text information.
StorageTypeURL A Uniform Resource Locator (URL) is a string of characters used to identify a web resource, such as a web page on an HTTP server, or a file on an FTP server.
TableCellInstance Represents the value of a table cell.
TableExtractMethod Defines settings and logic for a Data Section extraction method.
TableInstance Represents an instance of a Data Table object on a document.
TableProfile Defines settings for a Data Table which apply when processing a specific Content Type.
TableRowInstance Represents a table row in a TableInstance.
TrainingPage A Training Page represents an individual page which was used to train a Page Type. Training pages are not created or configured by users - they are created and managed internally by Grooper as documents are trained into a Content Model. Do not delete or edit Training Page objects except as directed by Grooper support staff.
UnattendedActivity Serves as the base class for all unattended activities.
VerticalLayoutProvider Detects cases where the array elements are arranged in a vertical column.
Visual Classifies documents or pages based on their visual appearance.
ZipHandler Mime type handler for Zip Archive files.
