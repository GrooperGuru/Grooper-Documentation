Grooper 2.70 Documentation

Grooper Installer and Initialization

The Grooper Installer installs Grooper.  Depending on the number of pre-requisites that exist on the target machine, the pre-requisites will take the majority of the installation time and may require one or more reboots.  The optional Microsoft SQL Express installer (used during the Grooper Wizard to install a local database instance), in particular, can take 15 minutes or more to install.

Grooper Installation Files

setup.exe - this is the main Grooper installation file.

Grooper (x##).msi - resource used by the Grooper installer. - x##: x86 for 32bit installer and x64 for 64bit installer

Setup.ini - resource used by the Grooper installer.

0x0409.ini - resource used by the Grooper installer.

program files folder - resource used by the Grooper installer.

Windows folder - resource used by the Grooper installer.

System32 folder - resource used by the Grooper installer.

System64 folder - resource used by the Grooper installer.

Installing Grooper

The section will describe the steps required to run the Grooper installer, configure Grooper via the installation wizard, and finalize the Grooper installation via Grooper Configuration.

 

Run the Installer

1.  Launch setup.exe

 

2.  Any required prerequisites will be prompted for installation at this time.  Click install to continue.  One or more system reboots may be required.

 

 

3.  After the prerequisites are installed, the Grooper InstallShield Wizard will install the Grooper files.  Click Next until the 'Ready to Install the Program' screen appears.  Click Install.

 

 

4.  After the Grooper InstallShield Wizard is Completed, click Finish and then launch Grooper Configuration.  

 

5.  This will prompt a Confirmation box confirming this is a new installation.  Click 'Yes' to continue to the Grooper Database Initialization Wizard.

 

 

Grooper Database Initialization Wizard

The Grooper Database Initialization Wizard can be used to connect to an existing instance of Grooper, connect to an existing SQL Server, or create a local database instance.  The Grooper Initialization Wizard also sets the primary File Store for Grooper for new installations and optionally sets the default service credentials for the local machine.

 

Navigation of the wizard is handled by the Next and Back buttons in the top left.  Users can only proceed forward (Next) if settings on the current panel validate correctly.  No actual configuration or changes will be applied until the Apply Configuration button is pushed on the final tab.

Service Credentials

 

1.  (Optional)  Enter administrative credentials that will be used as the default credentials for such things as running Grooper services, running Grooper License Servers, etc.  If left blank, these fields can be filled out during service setup.

 

2.  Click Next.

Database Connection

 

3.  If installing a Local Database Instance, click 'Install Local Database Instance'.  Otherwise, enter the Database Connection Parameters to connect to an existing SQL Server instance*.

 

4.  Click Next.

 

*If an instance of Grooper already exists in the selected database, the user will be prompted to connected to the existing database.  If 'yes' is chosen, Grooper will connect to that database instance,  the Initialization Wizard will exit, and Grooper Configuration will open.  

File Store

 

5.  Set a primary File Store for Grooper.  This is the location that Grooper files (scripts, temporary images, etc) will be stored.  The default location is 'C:\GrooperFileStore'.  This can be changed to any location to store Grooper files.  It is important to use a UNC or shared path if Grooper will be used on any other servers or workstations.

 

6.  (Optional) Encrypt the File Store - check the 'Encrypt Using Key Phrase' box and enter a key phrase.  This will encrypt any files stored in the File Store.

 

7.  Click Next.

Apply Configuration

8.  Click 'Apply Configuration' to apply the entered settings, create the Grooper database, initialize the Grooper Installation, and create the Grooper File Store.
