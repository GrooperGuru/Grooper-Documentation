Grooper 2.70 Documentation

Content Management System Connectivity

Grooper's integration with content management systems goes far beyond the "export only" integrations found in legacy capture solutions, providing powerful capabilities for both importing and exporting objects.

  • Automated Creation of Artifacts.  Grooper Content Models and Data Models can be created directly from the content types in a repository, eliminating the tedious, error-prone task of manually creating document types, fields, value lists in the capture environment which match those in the repository.  

  • Two-Way Document Interchange.  Grooper content types can be permanently mapped to content types in the repository, allowing ongoing two-way document interchange.  This capability allows Grooper to pull batches from a repository to perform additional data extraction or additional classification.  It allows Grooper's automated classification or data extraction to be trained from existing documents in a repository.  It also allows Grooper to be used for migrating content from one repository to another, optionally transforming the data in the process.

  • Repository Access.  The contents of a repository can be browsed and queried from Grooper Administration, allowing administrators to quickly and easily validate the success of exports, delete test batches, and etc. without exiting Grooper Administration.  Objects can also be copied and pasted between Grooper batches and the repository using standard clipboard operations.

Content Management Interoperability Services

Grooper supports the Content Management Interoperability Standard (CMIS), an open standard which allows applications to inter-operate with content management systems.  This standard, sponsored by AIIM, was originally approved in May of 2010, and is now support by a growing list of 30+ content management systems including Alfresco, EMC ApplicationXtender, EMC Documentum, HP Autonomy/Interwoven, HP Trim, HP Records Manager, IBM Content Manager, IBM Filenet, IBM Content Manager OnDemand, Microsoft SharePoint, OpenText, Oracle Webcenter, and many more.  A complete list of vendors supporting CMIS can be found here.

Grooper Implementation of CMIS

Grooper's implementation of CMIS is based on the following Grooper objects:

  • CMIS Connection. Represents a connection to a CMIS server.  Connections are specified in terms of a URL to the AtomPub binding of a CMIS server instance, a user name, and a password.  
  • CMIS Repository. - Represents a logical "repository" managed by a CMIS server.  For example, Microsoft's CMIS implementation for SharePoint exposes each SharePoint document library as a repository.  EMC Documentum exposes each Documentum repository as a repository, while EMC ApplicationXtender exposes each AX application as a separate repository.
  • CMIS Content Type.  Represents a content type within the repository, to which Grooper content types can be mapped.

Grooper can maintain connections to any number of CMIS servers running on any number of platforms.  The example below contains CMIS connections to Alfresco, EMC ApplicationXtender, Microsoft Sharepoint, EMC Documentum, and IBM FileNet.  Directly beneath each CMIS Connection object are the CMIS Repository objects for which connections have been configured.  

The general procedure for configuring a CMIS connection is as follows:

  1. Create a CMIS Connection using the AtomPub URL for the CMIS server.
  2. Use the Query Repositories command on the property panel for the CMIS Connection object to display a list of available repositories.
  3. Highlight the desired repository and use the Import Repository command on the property panel for the CMIS Connection object.
  4. Navigate to the newly-created CMIS Repository object (created as a child of the CMIS Connection).
  5. Select the content type(s) to be imported and use the Import Types command to create CMIS Content Type objects for each of the selected content types.
  6. Navigate to the newly-created CMIS Content Type object(s).  (CMIS Content Types are stored in the 'Content Types' subfolder of the CMIS Repository object.)
  7. Use the property panel for the CMIS Content Type to map it to a Grooper content Type, or use the Generate Content Type command to create a new Grooper content type from the CMIS Content Type.

Repositories Not Supporting CMIS

For organizations using systems which do not yet include support for CMIS, several options are available, as outlined below:

  • Grooper Standard Export - Grooper ships with a Standard Export script which can be used to export documents and metadata to a file system using industry-standard formats which can be imported to most document management systems.
  • Custom Export Scripts - If none of the advanced features of Grooper's CMIS interface are needed, a custom activity can be used to facilitate transfer of documents into the target repository.  Grooper development staff can be contracted to develop a custom export for the target repository.  It also is possible that a custom export script for the target repository has already been created.   Please contact your Grooper account representative to obtain additional information and pricing.
  • Custom CMIS Implementation - Grooper development staff can be contracted to develop a custom CMIS server application for the target repository.  Please contact your Grooper account representative to obtain pricing.
