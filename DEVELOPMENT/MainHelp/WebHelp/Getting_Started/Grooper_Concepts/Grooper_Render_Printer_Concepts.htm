Grooper Render Printer Concepts

Grooper Render Printer provides functionality for normalizing document content to PDF format so that it can be used with Grooper viewers and text extraction mechanisms.  Grooper Render Printer supports many different document content formats, as seen in the adjacent graphic.  An accurate representation of each document is generated by "printing" it through its native application.

 

Prerequisites and Concepts

The following section describes some of the key concepts and prerequisites for Grooper Render Printer.

 

Native Applications

As illustrated above, Grooper Render Printer supports many document content formats.  Each content format requires its native application (or equivalent) to be installed and configured on each Rendering machine under the user/service account that will executing Render activities.  Configuration includes, but is not limited to:  licensing, initial dialog interaction (e.g. Terms of Service agreements), etc.

 

System Settings

Grooper Render Printer requires the following three configurations for the logged in user/service that will be running Grooper Render Printer:

  • User Environment Variable - The Variable name must be set to "GROOPER_RENDERPATH".  The Variable value represents the temporary file path for rendering, by default this is "C:\Users\<logged-in user>\Documents\GrooperRender".
  • Default Printer - Grooper Render Printer must be set as the default printer in 'Printers and Faxes'.  This should happen automatically during the installation, but can be manually accomplished by going to 'Devices and Printers', right-clicking on "Grooper Render Printer", and selecting 'Set as default printer'.
  • Printer Name - It is not recommended to change the name/description of the Grooper Render Printer.  The name/description of the printer must contain "grooper render printer" for Grooper to recognize it as a Grooper Render Printer.
 

Resolution of Dialog and other User Prompts

Many applications are not inherently configured to support silent operations, such as printing.  Since Grooper Render Printer is typically run as a service, dialog boxes or other user prompts can potentially slow or disrupt render activities.  Grooper has many configurable timeouts and internal handlers to attempt to handle these issues, however additional steps can be taken to increase render efficiency:

  • Data Type - The Render Document activity has a configurable 'Dialog Response Data Type' property.  The 'Dialog Response Data Type' instructs Grooper to recognize and respond to specific dialog boxes and user prompts that may exist for different document content types.  For more details on configuring this Data Type, see Render Document.
  • Registry Settings - Many native applications have the ability to control or suppress dialog boxes and user prompts via registry settings.  Consult the Internet or documentation of each application to identify solutions to specific document content types.
  • Password Protected Documents - Grooper will detect and flag any password-protected documents.

 

Threading

Render Document is a single instance activity.  This means that Render Document can only run one thread on each machine.
