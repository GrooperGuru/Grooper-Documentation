Grooper 2.70 Documentation

Grooper Licensing

Grooper licensing is distributed to running applications by an instance of Grooper License Server, a web service which is installed using the Services Tab in Grooper Configuration.  The service can be configured to run on any available port number (the default port number is 13900).

 

At run time, applications must know how to communicate with the license server.  This is accomplished by configuring a License Server object in Grooper Administration which contains connection information for the running instance of Grooper License Server, and designating that License Server as the "active" license server by setting the License Server property of the Grooper Root object.  

Licensing Setup

The general procedure for installing Grooper licensing is as follows:

 

  1. In Grooper Configuration, use the Services Tab to install an instance of the License_Service on a server.  Remember the machine name and port number assigned to the service.

  2. In Grooper Administration, navigate to 'Infrastructure â€¢ License Servers' and create a new License Server object, configuring it with the machine name and port number for the service created in Step 1.

  3. Use the property panel of the newly-created License Server to install licensing using the serial number of license file provided with your purchase of Grooper.

  4. Navigate to the Grooper Root object and set the License Server property to the License Server object created in Step 2.

License Types

Grooper licensing includes the following types of licenses.

 

  • Server - The total number of concurrent servers which will run any Grooper application, including Administration and Dashboard.  A "server" is defined as any machine with a server operating system installed.
  • Concurrent Users - The total number of concurrent users who can be logged in to Grooper at any given time.  All attended processing modules acquire a concurrent user license when they connect to Grooper.  Grooper Administration, Grooper Configuration, and Grooper Dashboard do not require concurrent user licenses.
  • OCR Process - The total number of concurrent OCR operations which can be performed at any given time.  This includes all running threads for the Full Text OCR module, all running instances of the Auto Orient OCR IP command, all running threads for the Extract module performing zonal OCR, and all Scanner Profiles where the 'Auto Orient' option is checked.
  • Page Volume - The total number of pages which can be processed through Grooper over a given time interval. This includes all pages scanned or imported into Grooper, regardless of whether the associated batches are test batches or production batches.
