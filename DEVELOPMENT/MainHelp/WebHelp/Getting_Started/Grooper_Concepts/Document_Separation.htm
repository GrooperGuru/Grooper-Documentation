Document Separation

Document separation is the process of transforming a Batch of raw pages into a hierarchy of folders which represent individual documents.  Separation is most commonly needed when documents are acquired through scanning, or when raw pages are imported into Grooper.   This process is typically a precursor to classifying the folders as a content type.

Separation logic in Grooper is defined by a Separation Profile object, and can be applied in one of two ways:

  1. In real time during scanning, by specifying a Separation Profile in the settings of the Scan activity.
  2. Offline as a back-end process, by adding the Separation activity as a step in the Batch Process.

Separation profiles provide a variety of separation mechanisms, some of which can be used in real time during scanning, while others cannot.  Any separation methods which relies on results from the Full Text OCR activity cannot be executed in real time during scanning, as no OCR has been performed on the documents yet.  As such, these separation methods must be performed offline by including a Separation activity in the batch process.  The table below provides an overview of commonly-used separation use cases, along with notes on how they can be implemented.

Use Case Real Time Notes
Use patch codes, barcodes, or  fixed-page separation. Yes Select 'Events' separation method and define Separation Events which trigger separation.
Use Grooper Control Sheets. Yes Select 'Control Sheets' separation method.  Define and print Control Sheets.
Separate based on the visual detection of a shape Yes Select the 'Events' separation method, and define a Separation Event for each shape name which should trigger separation.  If running in real time, the IP Profile assigned to the Scan activity must include one or more Shape Detection commands configured to generate the appropriate shape names.  If running offline in the Separation activity, an Image Processing activity must be inserted into the batch process to run the Shape Detection commands beforehand.
Separate based on visual page classification Yes Enable real-time classification on the Separation Profile, ensuring that the Content Model assigned for real-time classification uses Visual classification.  Select the 'Events' separation method, and define a Separation Event for each content type which should trigger separation.
Separate based on lexical page classification No Insert Full Text OCR and Classify Pages activities into the batch process before Separation.  Select the 'Events' separation method, and define a Separation Event for each content type which should trigger separation.

Separate when a repeating value found on each page changes.

 No Insert a Full Text OCR activity into the batch process before Separation.  Define a Data Type to extract the value.  Select the 'ChangeInValue' separation method and select the defined data type.
Separate using page numbers found on the document pages. No Insert a Full Text OCR activity into the batch process before Separation.  Define a Data Type which returns a 1-based page number when executed on each page.  Select the 'EPI' separation method and select the defined data type.
Separate based on lexical rules No Insert a Full Text OCR activity into the batch process before Separation.  Define a Data Type which returns a value when separation should occur, and returns nothing when separation should not occur. Select the 'Pattern-Based' separation method and select the defined data type.
Using previous training, automatically separate a batch into documents. No Insert Full Text OCR and Classify Pages activities into the batch process before Separation. Select the "EspAutoSeparation" separation method.
