Data Instances

A Data Instance is a data structure used to represent data elements extracted from a document using Grooper's data extraction capabilities.  Data Instances are hierarchical in nature, and generally follow the structure of the Data Model they are associated with.  For example, performing data extraction using the data model shown below at the left will generate a hierarchy of Data Instances similar to what is shown below at the right.  Data Instances can be explored using the Instance Viewer control, available on the Instance Viewer tab of the Data Extraction Test Utility in Grooper Administration.

Data Model Extraction Results Data Instance

The root Data Instance in the hierarchy represents the entire content of the document.  Data elements extracted from the root of the document will perform their data extraction from the content of the parent element, generating Data Instances which are children of that element.  

At the Field level, each possible candidate for the field value is added as a child Data Instance of the field.  For example, if a date field is being extracted, and two dates appear on the document, there will be two candidates added below the field data instance.

Complex Data Instances

In a more complex Data Model where Sections and Tables are used, the hierarchy of Data Instances essentially maps out the structure of the document into a hierarchy of sections, tables, table rows, and fields, as shown below.

As mentioned previously, the root of the Data Instance hierarchy represents the entire content of the document.  Sections, Tables, and Fields at the root of the Data Model perform their extraction from the content of the parent element, which is the entire content of the document.

 

As Sections are identified and extracted, the content for each Section becomes the source from which its child data elements are extracted.  In the case of the Section "Semester 1" above, the content extracted for this section is shown below.  The "Year", "Semester", and "GPA" fields and the "Courses" table which are children of the section use only this content as the source for their data extraction, eliminating the rest of the document content from consideration.  

 

 

Likewise, each individual table row within the "Courses" table serves as the source from which the table column values "Course No", "Course Title", and "Grade" are extracted.

 

 

The hierarchical structure of Data Instances is central to Grooper's ability to model the data structure of complex documents.  By breaking down the document recursively into manageable units, the data extraction logic required for child elements is greatly simplified, as the unrelated content in the document is removed from consideration.

Navigating Data Instances in Scripts

It is common for scripts to need access to the data elements stored in a Data Instance.  For example, a Processing Script can be used to export complex data to a database, or a script could be added to the Data Model object to perform custom validation, perform custom integrations with line of business applications, or customize the user interface of the Data Review module.  To support such cases, Data Instances are designed to make data easily available to script writers.  

How to Find the Root Data Instance

Data Instances can be accessed through the IndexData member of the Batch Folder object.  The IndexData member gets initialized during classification, when a content type which defines a Data Model is assigned to the  Batch Folder object.  Prior to data extraction, the fields in a Data Instance are blank or set to their default values.  During data extraction, this blank instance is replaced by one populated with the extracted data elements.

 

A Processing Script which is running at the same folder level as the documents will receive ProcessTask events for each folder at that level.  The EventArgs for this event provides a "CurNode" member which contains a reference to the folder object.  The root DataInstance can be accessed as shown below.

 

Dim Folder As BatchFolder = e.CurNode

Dim RootInstance As DataInstance = Folder.IndexData

 

In the case of a script on a Data Model object, a DataInstance is passed to the following events as a member of the EventArgs:

  • AfterExtraction - e.Instance is set to the root DataInstance.
  • FieldEnter - e.Instance is set to the field's DataInstance.
  • FieldExit - e.Instance is set to the field's DataInstance.
  • Validate - e.Instance is set to the root DataInstance.
  • ValidateField - e.Instance is set to the field's DataInstance.

 

If field-level events which receive the field's DataInstance instead of the root DataInstance need to access the value of other data elements, Instance.Parent can be used to climb up the hierarchy of DataInstances.  

How to Access Data Elements

Once the root DataInstance has been acquired, data elements can be accessed easily through scripting as shown below.

 

    'Access the value of a root field:

    Dim StudentId As DataInstance = RootInstance.Fields("StudentID")

    Dim StudentIdValue As String = StudentId.Value

 

    'Loop through section instances for the 'Semester' section:

    For Each Semester As DataInstance In RootInstance.Sections("Semester")

 

      'Access field values inside the section:

      Dim Year As DataInstance = Semester.Fields("Year")

 

      'Get a reference to a table:

      Dim Courses As DataInstance.ITable = Semester.Tables("Courses")

 

      'Loop through the rows in a table:

      For Each Course As DataInstance.ITableRow In Courses.Rows

 

        'Access column values in a table row:

        Dim GradeValue As String = Course.Cells("Grade").Value

 

      Next

    Next

 

The interfaces ITable, ITableRow, and ITableCell are provided to simplify access to data stored in tables.  However, it is not necessary to use these interfaces, as the entire hierarchy can be navigated as DataInstance objects.

Modifying Field Values

Field values can be easily modified from scripts, as shown below.  Field values which are modified during the FieldEnter, FieldExit, or ValidateField event will be immediately reflected in the user interface of the Index Panel.  

 

Dim StudentId As DataInstance = RootInstance.Fields("StudentID")

StudentId.Value = "123456"
