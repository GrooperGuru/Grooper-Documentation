Reporting

Grooper's reporting subsystem is used to generate reports which provide insight into the efficiency of capture operations and the productivity of users who work in Grooper.  The product ships with a variety of pre-defined reports, and provides full support for the development of custom reports.

 

Reports are defined using a Report object.  Report objects contain the report definition, any scripting logic necessary to generate the report, and a set of properties which configure the report.  For example, the Pages Scanned report is configured with the following properties:

Report Instances are created using the Save_Instance command on the property panel for a Report object.  Simply set the report properties as desired, and then click the Save Instance command and enter a descriptive name for the Report Instance.

Viewing Reports

Reports can be viewed through a variety of mechanism in Grooper, as outlined below:

 

  • In Grooper Administration - When configuring settings for a Report, the report can be previewed in the property panel for the Report object by using the Run Report command.  Saved Report Instances can be viewed in the property panel for the Report Instance object.
  • In Grooper Dashboard - Grooper Dashboard can be configured with a list of Report Instances to be displayed in tabs at the bottom of the main window.
  • In Grooper Kiosk - The Grooper Kiosk application displays full-screen reports and is designed to be used with a wall-mounted display screen to display production statistics in a production capture operation.  The application is configured with a list of Report Instances which it will repeatedly cycle through.

Predefined Reports

The following predefined reports ship with Grooper.

Data Capture Automation

This report measures the automation rate of data extraction by displaying the percentage of records where a manual edit was required in the Data Review module.

 

 

The properties for this report are outlined below.

 

Property Name Description
Report Range
Range Type Controls the period of time for which the report is run.  Can be one of the following values:
  • Specific Dates - Use the specific dates provided in Start Date and End Date.
  • Today - Runs report for the current date.
  • Yesterday - Runs report for the current date minus one day.
  • MTD - Runs report for the start of the current month to the current date.
  • YTD - Runs report for the start of the current year to the current date.
  • Past7Days - Runs report for past 7 days.
  • Past30Days - Runs report for past 30 days.
  • Past60Days - Runs report for past 60 days.
  • Past90Days - Runs report for past 90 days.

 
Start Date If Range Type is set to 'SpecificDates', the start date for the report.
End Date If Range Type is set to 'SpecificDates', the end date for the report.
Report Settings
Batch Process If set to a value, restricts the report to the indicated Batch Process.

Scan Operator Efficiency

This report measures the overall efficiency of scan operators by comparing the actual speed at which the scanner is running to the effective speed of the operator after the delays associated with out of paper conditions, paper jams, double feeds, and idle time are accounted for.  Essentially, this report measures the percentage of time spent with a batch open where the scanner is actually feeding documents.  The more time an operator spends without the scanner running, the lower the efficiency rating.

Kiosk Reports

The Kiosk Reports folder contains variations of the reports listed above which are formatted for the 16:9 aspect ratio of the standard wide-screen display.  These reports are designed to be used with the Grooper Kiosk application.

Custom Reports

Grooper's reporting subsystem is based on Microsoft Reports, and reports are created using the Report Designer in Microsoft Visual Studio through Grooper's scripting interface.  The script for report generation is defined on the Report object.  For information on how to create a custom report, refer to the documentation for the Report object, or the How To Guide.
