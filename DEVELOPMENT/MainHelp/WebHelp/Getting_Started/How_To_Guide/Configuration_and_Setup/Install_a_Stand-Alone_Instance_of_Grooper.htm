Install a Stand-Alone Instance of Grooper

This guide discusses how to install a stand-alone instance of Grooper. A stand-alone instance is one which typically only involves a single workstation, and uses a local instance of SQL Server Express installed from the Data Sources Tab of  Grooper Configuration.  

Step 1: Install Grooper

  • Run the Grooper Installer.

Step 2: Run Grooper Configuration

  • Launch Grooper Configuration.
  • When the confirmation prompt shown below appears, click Yes.  This will launch the Database Initialization Wizard.

 

Step 3: Start the Database Initialization Wizard

  • Click Next on the Welcome screen.

 

Step 4: Specify Default Service Credentials

  • Enter the default service credentials.  (Optional)  This is the user account under which services will be installed by default.  Setting this value prevents the information from having to be re-entered each time a service is installed using the Services Tab of Grooper Configuration.

 

Step 5: Install SQL Server Express

  • On the Database Connection screen, click 'Install Local Database Instance'.
  • This will launch the installer for the SQL Server Express 2014.  This installation can take several minutes.

 

Step 6: Review and Accept Database Connection Settings

  • Once the installation of SQL Server Express has completed, the connection settings will be populated automatically, as shown below.  
  • Click Next to test the connection to the database and move to the next step.

 

Step 7: Set the Storage Path

  • Set the storage path to be used by Grooper for content files.  If this instance of Grooper is only to be accessed from the local machine, a local path can be used.  Otherwise, a UNC path must be used.
  • Click Next.

 

Step 8: Apply the Configuration

  • Click the Apply Configuration button to start the database initialization process.
  • Once completed, your new instance of Grooper is ready to use.  Launch Grooper Administration to validate that Grooper is now functional.
