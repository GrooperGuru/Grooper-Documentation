Install a Processing Module as a Service

This guide demonstrates how to install a processing module such as Full Text OCR, Image Processing, Extract, and etc. as a Windows service.

Step 1: Launch Grooper Configuration

  • On the workstation or server where services should run, launch Grooper Configuration.
  • Select the Services Tab.

Step 2: Add a Service Instance

  • Click the Add button.  This will display the 'Select Service Type' dialog, a shown below.
  • Select 'Processing Module Service' and click OK.

 

Step 3: Configure the Service Instance

  • The Service Instance Configuration dialog box shown below should now be visible.
  • Select the Processing Module to be hosted by the service instance.

  • If the processing module is to run on a workstation where users will be working, set the Thread Priority to 'BelowNormal'.  This will minimize the performance impact on the user by ensuring that the user interface thread always has higher priority than the background worker threads.

  • Enter the Number of Threads the service instance should run.  This should be less than or equal to the number of processing cores on the machine.

  • Enter the user name and password of the account under which the service should run.  (This may be already set if the Settings Tab has been configured with default service account credentials.)

  • When finished, click OK.

 

Step 4: Start  the Service

  • Wait for the service installation to complete.
  • Select the new service instance from the list and click 'Start Service'.
