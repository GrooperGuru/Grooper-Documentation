Grooper 2.70 Documentation

Estimate System Sizing Metrics

When deploying a new instance of Grooper, it is important to design the system with enough server processing cores, file storage space, and database storage space to accommodate the volume of documents to be processed.  This topic provides information to help IT planners estimate these metrics.

Server Processing Cores

The number of server processing cores required to process a specific volume of documents in Grooper is dependent on many factors:

 

  • The resolution, color depth, physical size, and condition of the documents to be processed.
  • The number and type of steps included in the Batch Process.
  • The average number of pages per document.
  • The number of document types in the Content Model.
  • The number of data elements to be extracted.
  • The responsiveness of external databases and content repositories.
  • The speed of file storage devices.
  • The clock speed of the CPU cores.
  • Etc...

 

Testing with actual batches running through the actual Batch Process to be used is the only sure way to determine how many processor cores will be required, but it is possible to estimate the number of cores required using metrics derived from testing on documents of various sizes and color depths.  

 

For example, in a typical batch process, the two modules requiring the most processing time are Image Processing and Full Text OCR, as these two modules operate at the page level, performing work on every individual page in each batch.  On a 3.0 GHz processor core, these modules demonstrate the following performance using the Standard Full Text OCR Profile and the Default Scan Cleanup IP Profile.

File Storage Space

The amount of space required for storing files also varies greatly depending on system configuration.  The following factors have the largest impact on file storage requirements:

 

  • The resolution and color depth at which documents are scanned.
  • The quality of the documents (noisy images don't compress as well as clean images).
  • The percentage of pages on which Full Text OCR is being performed.
  • Whether the Undo Image Cleanup feature is enabled.
  • Whether alternate images are being generated to support the OCR process.
  • The amount of time batches are retained in Grooper after export.
  • Etc...

Database Storage Space

Database storage requirements are dependent on many factors also.  The most prominent factors affecting database storage requirements are as follows.

 

  • The average number of pages stored in Grooper at any given time.
  • The number of pages trained for classification purposes.
  • The number of fields trained for data extraction purposes.
  • The average number of pages per folder in batches.
  • The logging settings configured on the Grooper Root object.
  • Etc...

 

With that said, a good general rule of thumb for Grooper database sizing is around 100 Kb per page object.  Applying this rule to a scenario where an average of 5X the daily processing volume is stored in Grooper at any given time will produce the following total database storage estimates:
