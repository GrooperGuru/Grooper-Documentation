HOW TO: Configure Visual Classification

Visual classification classifies documents based on how they look, using features generated by the Extract Features IP command.  It differs from Lexical classification in that it does not require full text OCR, and can be used in real time during scanning to separate and classify structured documents.

 

Visual classification can be enabled through the following steps:

 

  1. Create a Content Model which defines the document types to be classified.
  2. Define an IP Profile containing an Extract Features command.
  3. Set the Classification Method property of the Content Model to 'Visual'.
  4. Set the Image Features IP Profile property of the Content Model to the IP Profile created in step 1.
  5. Train the Content Model with one or more examples of the document type.

 

To use visual classification in real time during scanning, use the following steps:

 

  1. Create a Separation Profile.
  2. Set the Enable Page Classification property to true.
  3. Set the Content Model property to a Content Model which uses visual classification.
  4. Add a Separation Event to the Separation Profile, and set the Event Type property to 'ContentType'.
  5. Set the Trigger Content Type property to the document type which should trigger separation; or, if many document types are in use, select the parent content type to enable all child content types to trigger the event.

  6. Assign the Separation Profile to the Scan Module.
