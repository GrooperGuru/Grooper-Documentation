Perform Database Lookups

Grooper provides a simple database lookup mechanism which requires no scripting.  This feature is only useful in situations where the lookup is based on the value of a single field.  More complex lookups should be implemented through scripting.

 

This exercise will assume we are using a Data Model with the fields shown below.  The lookup will be performed on the PO Number field.  If to matching database row is found, the PO Number field should display an error.  If it is found, then the Vendor No field should be updated with the Vendor No from the database table.

 

Step 1: Create a Data Connection

  • In Grooper Administration, navigate to Infrastructure â€¢ Data Connections.
  • Create a new Data Connection object, configured to connect to the database where the lookup data is stored.
  • Click the Test Connection button to test the connection.  A list of all tables in the database should now be displayed.

Step 2: Import a Reference to the Database Table

  • Select the table containing the lookup data, and click the 'Import Command'.
  • Expand the Data Connection in the tree structure, and verify that a new Database Table object has been created as a child of the Data Connection object.

Step 3: Configure the PO Number Field

  • Expand the Data Model and navigate to the Field which will be used as query criteria.  In our example, this would be the PO Number field.
  • Set the Database Table property of the Field to the Database Table object created in Step 2.
  • Go to the Mappings Tab.  Map the PO Number field and the Vendor No fields to their corresponding fields in the database table.

 

Database lookups should now be active, and can be tested using the Data Extraction Test Utility.
