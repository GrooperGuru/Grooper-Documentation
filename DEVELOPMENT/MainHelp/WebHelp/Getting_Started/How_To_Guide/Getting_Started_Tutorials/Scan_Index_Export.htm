HOW TO: Scan, Index, and Export

This tutorial provides a walk-through of the process required to build a simple batch process.  The process created in this tutorial will perform basic image cleanup during scanning, and real-time separation based on Patch T patch code sheets.  The documents will be indexed manually, and exported to a file system directory in PDF format, with the metadata exported as XML.

Step 1 - Create a New Content Model

The first step will be to create a Content Model, which will describe the document types and data elements we will be capturing.

 

  • In Grooper Administration, navigate to the Content Models folder.
  • Right-click the Content Models folder and select "Add Content Model"
  • Name the IP Profile 'Scan Index Export' and click OK.

Step 2 - Add a Document Type

We will need at least one document type, so let's create that first.

 

  • Right-click the 'Scan Index Export' Content Model and select 'Add Document Type'.
  • Name the new Document Type 'Generic Document' and click OK.

Step 3 - Add a Data Model to the Content Model

As we will be indexing field values, we need to add a Data Model to the Content Model so that we can define fields.

 

  • Select the 'Scan Index Export' Content Model in the left pane.
  • Click the Create Data Model toolbar button on the property panel for the Content Model.
  • The Content Model should now look like this:

Step 4 - Add Index Fields

This step will add the fields we will be indexing.  To keep things simple, we will just add two fields: Description and Document Date.

 

  • Right-click the newly-created Data Model immediately below the 'Scan Index Export' Content Model, and select 'Add Data Field'.
  • Name the field 'Description' and click OK.
  • Right-click the Data Model again and select 'Add Data Field'
  • Name the field 'Document Date' and click OK.
  • In the property panel for the 'Document Date' field, change the Display Width property to 100, and set the Data Type property to the 'Data Types • (system) • Date' Data Type.
  • Click the Save button.  The Content Model should now look like this:

Step 5 - Create an IP Profile for Scan-Time Image Cleanup

The first step will be to create an IP Profile containing image cleanup commands, which will be used for real-time image cleanup during scanning.  

 

  • In Grooper Administration, navigate to Profiles • IP Profiles.
  • Right-click the IP Profiles folder and select "Add Ip Profile"
  • Name the IP Profile 'Scan Cleanup' and click OK.
  • Add an Auto Deskew command.  In the property panel for the new IP Profile, click Add • Image Transforms • Auto Deskew.
  • Add an Auto Border Crop command.  In the property panel for the new IP Profile, click Add • Image Cleanup • Auto Border Crop.

Step 7 - Create a Batch Process

This step will create a Batch Process to specify the steps through which batches will be processed.  We will be adding 3 steps: Scan, Data Entry, and Export.

 

  • In Grooper Administration, navigate to Batch Processing • Processes • Working.

  • Name the Batch Process 'Scan Index Export' and click OK.

Step 8 - Publish and Test the Batch Process

Your new process is now complete...time to publish and test it.

 

  • Use the Publish command on the toolbar to publish the Batch Process
  • Launch the Scan Module and create a batch using the 'Scan Index Export' Batch Process.
  • Scan some documents.
  • Complete the batch.
  • Launch the Data Review Module and click Start.
  • Enter data for all documents.
  • Complete the batch.
  • Launch Grooper Script and click start.
  • When all tasks have been processed, navigate to the file system directory configured for the export step and verify that the documents and metadata have been exported.
