Modify the Flow of a Batch Process

In some cases, it may be necessary to dynamically modify the flow of a Batch Process.  This can be done easily using Grooper's Scripting interface.  

 

For example, consider the Batch Process shown below, which contains a step called Rescan.  When a batch comes out of Image Review, if there are pages flagged in the batch, we want the batch to go to Rescan.  If not, we want the batch to proceed to the OCR step.

 

Step 1: Add a Script to the Batch Process

  • In Grooper Administration, go to the Scripting Tab for the Batch Process.
  • Click the Create button.  

Step 2: Add Code to the Script

  • Click the 'Edit in Visual Studio' button.
  • When Visual Studio appears, open ScriptingSession.vb
  • Create a handler for the BatchCompleting event of the BatchProcess object, and enter the following code.

 

  Private Sub BatchProcess_BatchCompleting(sender As Object,

      e As BatchProcess.BatchCompletingEventArgs) Handles BatchProcess.BatchCompleting

    Select Case e.StepCompleted.Name

      Case "Image Review"

        If (e.Batch.HasFlaggedPages) Then

          e.NextStep = BatchProcess("Rescan")

        Else

          e.NextStep = BatchProcess("OCR")

        End If

      Case "Rescan"

        e.NextStep = BatchProcess("Image Review")

    End Select

  End Sub

Step 3: Save and Compile

  • Exit Visual Studio and save the project.
  • Click the Compile button.

 

As shown in this exercise, setting the NextStep property of the BatchCompletingEventArgs allows the next step to be set to any step in the Batch Process,
