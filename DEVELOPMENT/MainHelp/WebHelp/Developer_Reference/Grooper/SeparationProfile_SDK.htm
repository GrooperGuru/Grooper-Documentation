SeparationProfile

Separation Profiles are used to specify how pages will be separated into documents and folders within a batch.

Method SeparationMethod Enables Control Sheet processing for this Separation Profile. Can be one of the following values:
  • Events: Separate the batch based on Separation Events. Separation events can separate based on patch codes, barcodes, blank pages, detected shapes, or page content types.
  • ControlSheets: Separate the batch using Grooper Control Sheets.
  • ChangeInValue: A new folder is created each time the extracted value changes.
  • PatternBased: A new folder will be created if the extractor produces any result on the page.
  • EPI: Performs separation using embedded page information (EPI). The provided extractor should be a Data Type which returns a 1-based integer page number. When this binding method is in use, Grooper applies the following logic: 1) If the page number is 1, create a new folder. 2) If the page number is > 1, it will be appended to the current folder only if it is in sequence with the last recorded page number. If the page is out of sequence, it will be left as a loose page.
  • EspAutoSeparation: Performs separation using ESPâ„¢ automatic separation. Requires that classification has previously been run at page level.
  • UndoSeparation: Flattens a Batch or Batch Folder by recursively removing all children Batch Folders leaving only loose pages. All pages will retain their original postion.
MissDisposition MissDispositionEnum When the provided extractor produces no values for the page, the page will be handled based on the value of this property. Can be one of the following values:
  • LeaveAsLoosePage: Leave the page as a loose page.
  • AppendToLastFolder: Append the page to the last folder created. If no folders have been created, the page will be left as a loose page.
Name String The name of the node. Names must be unique within the parent container node. Use DisplayName when displaying in a user interface.           The type of event to log. A combination of the following flags:
  • Err: Errors - Critical error messages.
  • Wrn: Warnings - Non-critical warning messages.
  • Prc: Procedural - Tracing of procedural calls. NOTE: Enabling this option can generate large volumes of log entries. Use only during troubleshooting.
  • Inf: Information - Detailed information logging. NOTE: Enabling this option can generate large volumes of log entries. Use only during troubleshooting.
  • Aud: Audit - Audit trail events such as batch creation, object deletion, etc.

          The message, or message format. Uses String.Format syntax.
