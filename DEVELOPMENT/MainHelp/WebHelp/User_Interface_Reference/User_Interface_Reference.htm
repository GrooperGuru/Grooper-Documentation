Grooper 2.70 Documentation

User Interface Reference

The following sections provide documentation for individual Grooper client applications as well as links to all of the Grooper Control and Form UI Elements.

Name Description
Grooper Administration Grooper Administration is a browser for the Grooper Tree Structure, used by administrators to configure and manage objects in a Grooper repository. This application uses an explorer-style view, where the tree structure is displayed in a panel on the left, and a panel on the right displays the properties of the currently selected node.
Grooper Configuration Grooper Configuration provides a user interface for configuring machine-specific settings on a Grooper workstation or server. This application is used to perform the following administrative tasks:
  • Creating and managing database connections from the local machine to Grooper repositories.
  • Installing a local instance of SQL Server Express to accommodate stand-alone installations of Grooper.
  • Initializing new Grooper databases.
  • Installing and managing services which will run on the local machine.
  • Configuring various settings which apply to the local machine.

In general, Grooper Configuration is used to perform configuration work which is specific to a workstation or server, while [Grooper Administration] is used for configuring objects in the Grooper Tree Structure which apply to all workstations and servers. After installing a new Grooper workstation or server, run Grooper Configuration to connect it to a Grooper repository.
Grooper Dashboard Grooper Dashboard provides batch management and reporting capabilities for production batches. This application is used by managers, supervisors, and end users to view and manage production batches as they are processed through Grooper Batch Processes. Typical uses for Grooper Daskboard include the following:
  • Monitor system throughput and user productivity.
  • Create, edit, or delete batches.
  • Import batches of documents into Grooper using an Import Provider.
  • View reports and analytics.
  • Identify and resolve batch processing errors.
  • Start, stop, or edit Grooper services.
Grooper Kiosk Grooper Kiosk provides kiosk-style display of reports. Designed to run in full screen mode on a wall-mounted display, Grooper Kiosk can be used to provide real-time analytics feedback to workers in production document processing centers. Please note that it may be necessary to set the Windows Taskbar to Auto Hide mode, in order to remove the start menu and task bar from the display screen.
Grooper Attended Client Module Grooper Attended Client provides a user interface for all Attended Activities. BatchView controls will be displayed in the current panel while processing. On the right is a viewer control for displaying specific batch pages in detail.
There are multiple ways that this application can be opened:
  • Case 1: Opened from BatchManager, a BatchId command line parameter will be present. In this case, the application is closed automatically when the final task for the Batch is completed, the task is closed (user clicks the 'Stop Processing' button), or the batch is deleted.
  • Case 2: Opened using a shortcut, installed by the installer. An ActivityType command line parameter will be present. In this case, the application sets a filter for the passed in ActivityType that cannot be modified. The application does not close automatically.
  • Case 3: Opened manually. In this case, the application can be used to process any task and does not close automatically.
Grooper Unattended Client Module Grooper Unattended Client performs production task processing in a desktop application. Provides an alternative to running production tasks using the Task Processing service.
