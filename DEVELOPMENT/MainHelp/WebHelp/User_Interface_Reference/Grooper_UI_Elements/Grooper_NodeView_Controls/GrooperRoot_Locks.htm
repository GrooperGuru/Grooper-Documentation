GrooperRoot Locks NodeView

Provides a user interface for administrators to review locked objects, and manually unlock them when needed. Grooper locks objects in the repository during update operations to synchornize the activity of multiple processes or threads. In some cases, such as a database outage or system error, an object may become stuck in locked state. When this occurs, administrators can find and clear these locks using the Lock Manager Control.

Toolbar Commands

Command Name Description
Go To Related Object Navigates to the selected node.
Refresh Refresh the list of active locks.
Remove Lock Unlock the selected item(s).
