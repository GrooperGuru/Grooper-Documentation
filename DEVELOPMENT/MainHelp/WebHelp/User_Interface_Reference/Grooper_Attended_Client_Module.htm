Grooper Attended Client Module

Grooper Attended Client provides a user interface for all Attended Activities. BatchView controls will be displayed in the current panel while processing. On the right is a viewer control for displaying specific batch pages in detail.

There are multiple ways that this application can be opened:
  • Case 1: Opened from BatchManager, a BatchId command line parameter will be present. In this case, the application is closed automatically when the final task for the Batch is completed, the task is closed (user clicks the 'Stop Processing' button), or the batch is deleted.
  • Case 2: Opened using a shortcut, installed by the installer. An ActivityType command line parameter will be present. In this case, the application sets a filter for the passed in ActivityType that cannot be modified. The application does not close automatically.
  • Case 3: Opened manually. In this case, the application can be used to process any task and does not close automatically.

Child Controls

Control Type Description
Batch Multi View A container control that hosts one or more child controls derived from BatchView. If more than one child control is to be hosted, then the child controls are automatically added to a TabControl to provide a convenient tabbed UI for switching between the child controls.
Data Display Timer Displays statistics on a timer, typically while a background process is running.

Main Menu Commands

  Command Name Shortcut Keys Description
Batch / Create... Alt+N Create a new Grooper batch.
Batch / Delete Alt+D Delete the current Grooper batch.
Batch / Properties... Alt+P Display Properties of the current Batch
  Help / About... Display the About dialog box
Help / Show Help... Display Grooper Help.
Processing / Complete Task F9 Complete the current task.
Processing / Exit Alt+F4 Exit Attended Client.
Processing / Open Task... Ctrl+O Open a single existing task and process it.
Processing / Process Tasks F5 Start processing existing tasks using timer.
Processing / Settings... Alt+S Display the Attended Client options dialog.
Processing / Stop Shift+F5 This does not actually stop the status timer, which continues to run It merely sets a flag determining if the polling timer automatically processes the next available task or not.
