Grooper 2.70 Documentation

Upgrade Notes

Notes for upgrading from version 2.60 to version 2.70

Procedure for Upgrading (Recommended):

  1. Stop all Grooper services.
  2. Run the Upgrade by clicking Continue.
  3. Recompile all scripts from the Scripts tab on the Root Node in Grooper Administration.
  4. Restart Grooper Services.
  5. Pause and resume all Batches with a Status of Ready or Working. Note: Batches in a ready or working state before the upgrade may no longer process correctly after the upgrade unless they are first paused and resumed.

Architectural Changes:

  • Created new assemblies and reorganized the location of many classes. This could effect existing scripts. The following new assemblies have been created:
    • Grooper.Core
    • Grooper.CMIS
    • Grooper.IP
    • Grooper.OCR
    • Grooper.Capture
    • Grooper.Reporting
  • Redaction Profiles - Redaction Profiles have been removed from Grooper. The settings of existing Redaction Profiles have been applied directly to any Redaction Activity that was referencing a Redaction Profile.
  • IP Profile Changes
    • IP Commands have been converted to IP Steps.
    • IP Command Execution Conditions have been converted to IP Step Should Execute Expressions.
    • Notable settings changes (while an effort has been made to accurately transfer applicable settings, it is recommended that the following IP Steps are reviewed and tested before production use):
      • Auto Border Crop
      • Blob Removal
      • Border Fill
      • Crop
      • Document Despeckle - No longer has a 'Region of Interest' property.
      • Edge Dectection
      • Extract Features
      • Filter
      • Halftone Removal - No longer has a 'Blank Out Regions' property. Several new properties are available for accurately identifying halftone regions.
      • Hole Punch Removal
      • Inverted Region Dropout - (formerly Invert Dark Regions) All properties have changed.
      • Line Detection
      • Line Removal - Max Entropy property removed - no longer required.
      • Median Noise Filter - This has been replaced with Filter. However, if Median Noise Filter was being used for Despeckle, it may be better to use Speck Removal.
      • Read Barcodes - Added support for Truncated PDF417.
    • Internal Type Name changes. All IP Commands have had the 'Ip' prefix and 'Command' (or 'Cmd') suffix removed. The following have also had been renamed and/or converted:
      • IpAutoOrientOcrCommand is now 'AutoOrient'.
      • IpLineDetectHoughCommand is now 'LineDetect'.
      • IpAutoCropCommand is now 'AutoBorderCrop'.
      • IpCropBordersCommand is now 'Crop'.
      • IpAutoNegateCommand is now 'NegatedRegionRemoval'.
      • IpAutoInvertTextCommand is now 'NegatedRegionRemoval'.
      • IpInvertDarkRegionsCommand is now 'NegatedRegionRemoval'.
      • IpDocumentDespeckleCommand is now 'SpeckRemoval'.
      • IpChangePixelFormatCommand is now 'Convert'.
      • IpAutoColorDepthCommand is now 'AutoConvert'.
      • IpGrayscaleDepthCommand is now 'AutoConvert'.
      • IpRandomizeImageDefectsCommand is now 'RandomizeDefects'.
      • IpScanLineRemovalCommand is now 'LineRemoval'.
      • IpBrightnessHistgramStretchCommand is now 'Histogram'.
      • IpMedianNoiseFilterCommand is now 'Filter'.
      • IpImageAnalysisCommand is now 'AutoQA'.
    • The following commands have been deleted and do not have an upgrade path:
      • IpDifferenceOfGaussianCommand
      • IpDocumentOutlineCommand
      • IpDocumentThinningCommand
      • IpAutoBitDepthCommand
  • Grooper Reporting - Grooper System Reports have been converted from scripts to native Grooper Objects. This new architecture will make it easier to create custom reports. In order to preserve script code, existing non-system reports will be moved to Infrastructure\Object Libraries\Legacy Reports. These custom reports will ultimately need to be re-written using the new Report Definition or Simple Report Definition base classes.
  • Render Document Activity - Lexicon properties (FileExtensionMappings and ExcludedFileExtensions) converted to EmbeddedLexicons.
  • File Import (ImportProvider) - Lexicon properties (AllowedFileExtensions and ExcludedFileExtensions) converted to EmbeddedLexicons.

Miscellaneous:

  • System objects have been removed from Grooper installations. The most updated version of system objects for each Grooper version can be found on Grooper Central (**grooper central url**).
