Grooper 2.70 Documentation

Index Navigator

Displays Document Instance objects, providing a user interface for reviewing and editing the data elements associated with a document. The Index Navigator Panel is a hierarchy of Index Panel Control objects, designed to allow editing of the hierarchical Data Instance objects produced by the Extraction activity.

Inherits from: Batch View

Properties

Property Name Description
General
Flag Invalid Documents
Type: Boolean, Default: False
When set to True, documents with invalid index data will be flagged.
Processing Level
Type: ProcessingLevel, Default: Level1
The level at which documents are expected to occur in the batch relative to the Batch Root. Can be one of the following values:
  • Level1
  • Level2
  • Level3
  • Level4
  • Level5
  • Level6
  • Level7
  • Level8
For example, Level1 represents folders that are direct descendants of the Batch Root. Level2 represents children of the Level1 folders.
Display Parent Folder(s)
Type: Boolean, Default: True
Controls whether or not parent folders are viewable/editable while processing. For more information see Levels.
Auto-Load Next Invalid Document
Type: Boolean, Default: True
If set to True, after tabbing out of the last field of a document, the next invalid document will be loaded. If set to False, the next available document, regardless of status, will be loaded.
Flag Messages
Type: Embedded Lexicon
A list containing pre-defined flag messages. (See Embedded Lexicon)
