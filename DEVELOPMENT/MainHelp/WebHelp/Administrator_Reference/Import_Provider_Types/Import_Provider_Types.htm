Grooper 2.70 Documentation

Import Provider

Import providers enable import of file-based content from a variety of sources such as file systems, mail servers, and content repositories. Import providers can be used in two ways:
  • To perform user-directed imports from the Batch Manager control.
  • To perform scheduled imports using the Import Watcher service.

List of Import Provider Types

Name Description
  CMIS Import Provides import connectivity to CMIS-compliant content management platforms. CMIS Import relies on settings defined by CMIS Content Type objects. As such, it is important that the relevant CMIS type has been imported, and that a corresponding Grooper Content Types has been created and mapped to it.
  File System Import Provides import connectivity to file systems. See the File System Action activity for follow-up actions which can be performed in a Batch Process.
  FTP Import Provides import connectivity to ftp servers.
  Mail Import Provides mail message import capabilites over the IMAP protocol. Mail messages are imported to Grooper as Electronic Documents in RFC822 (https://www.ietf.org/rfc/rfc0822.txt) format. The Render Document activity can be used to generate a PDF version of the mail message and its attachments. This process will make the mail content viewable in Grooper. PDF Text Extract can then be used to generate OCR data, exposing email content to the full suite of Grooper classification and data extraction technologies. The Mail Action activity can be used to perform a variety of actions on mail messages, including metadata and attachment extraction.
  SFTP Import Provides import connectivity to SSH/SFTP servers.
