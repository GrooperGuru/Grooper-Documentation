Data Element Profile

Data Element Profiles store settings for Data Elements such as sections, tables, and fields on a Content Type. Data Element Profiles are created and managed on the "Data Element Profiles" tab of any Content Type, using the Data Element Profile Manager control. They allow settings for data elements to be customized for a specific document type. For example, a Data Element Profile could be used to make a field read-only on Document Type A, but leave it editable on Document Type B.

List of Data Element Profile Types

Name Description
Field Profile Defines settings for a Data Field which apply when processing a specific Content Type. Field Profiles are created on Data Element Profiles tab of a Content Type, and are generally created for one of two purposes:
  • To override settings for a Data Field on a particular Content Type.
  • To define zonal extraction parameters for a Data Field on a particular Content Type. Once an Field Profile has been created, zones may be defined as children of the Field Profile.
Section Profile Defines settings for a Data Section which apply when processing a specific Content Type.
Table Profile Defines settings for a Data Table which apply when processing a specific Content Type.
