OMR Zone

OMR Zones are used for Optical Mark Recognition (OMR), to read the state of checkboxes and determine if they are checked or unchecked. OMR Zones work by measuring the black pixel ratio in each zone.

There are 3 possible configurations for OMR zones:
  • Single Checkbox - Create a single OMR zone. Tune the OMR Threshold and specify a negative value and a positive value.
  • Check One - Define a list on the Data Field containing an entry for each possible output value, and define an OMR zone for each checkbox. Set the Zone Processing property of the Field Profile to BestConfidence. Set both the OMR Negative Value and the OMR Positive Value to the list value associated with the checkbox.
  • Check All That Apply - Create an OMR zone for each checkbox. Set the Zone Processing property of the Field Profile to BestConfidence, and define a Separator String if desired.
OMR Zones are created as the child of an Extraction Profile. They can have Registration Zones as children.

Inherits from: Extract Zone

Properties

OMR Threshold
Type: Double, Default: 10%, Range: 0% - 100%
The percentage of black pixels above which an OMR zone is considered filled.
OMR Positive Value
Type: String, Default: true
The value to be stored in the field when an OMR read is positive. The variable '{Ratio}' can be inserted to return the actual black-white pixel ratio of the OMR zone.
OMR Negative Value
Type: String, Default: false
The value to be stored in the field when an OMR read is negative. The variable '{Ratio}' can be inserted to return the actual black-white pixel ratio of the OMR zone.

OMR Negative Value

Target List Value

The value to be stored in the field when this OMR zone is the most confident.

Page No

The page number associated with this zone. A negative value indicates on offset starting at the end of the document. For example, -1 indicates the last page of the document -2 indicates the 2nd to last page of the document, and etc.
Bounds

The location of the zone.
