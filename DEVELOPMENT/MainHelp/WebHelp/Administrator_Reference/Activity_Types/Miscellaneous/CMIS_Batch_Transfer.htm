CMIS Batch Transfer
Transfers a Batch to another Grooper repository, optionally launching the batch into a Batch Process in the target repository. This activity is designed to facilitate distributed applications, providing a secure mechanism for moving batches between Grooper repositories over web protocols. To use this activity, the following prerequisites must be met:
  • An instance of Grooper CMIS Server must be installed on the target repository.
  • A CMIS Connection must be created in the source repository, which connects to the Grooper CMIS Server discussed above.
  • The CMIS Repository representing the target repository must be imported in the source repository.


Inherits from: Unattended Activity

Properties

Property Name Description
General
Target Repository
Type: CMIS Repository
The Grooper repository to which the batch will be transferred. The CMIS Repository selected must represent a connection to a Grooper repository. (See CMIS Repository)
Target Step
The Batch Process Step at which the Batch will be started. If a step is selected here, the batch will be launched into the associated Batch Process, starting at the specified step.
Target Batch Process
Type: String
The Batch Process which contains the selected step.
Target Batch Status
Type: BatchStatus, Default: Ready
The Batch status for target batches after transfer is complete. Can be one of the following values:
  • Ready
  • Paused
Delete Batch
Type: Boolean, Default: False
Specifies whether the batch on the local system should be deleted after successful migration.
Error Handling
Error Disposition
Type: IssueDisposition, Default: Flag, Log
Determines what happens when an error occurs processing an activity. A combination of the following flags:
  • None: The issue will be ignored, and the item will complete successfully.
  • Flag: The associated Batch Folder or Batch Page will be flagged.
  • Log: The issue will be logged to the Grooper log.
  • Stop: The Batch will be set to error.
Maximum Consecutive Errors
Type: Int32, Default: 0
The maximum number of consecutive errors, after which a critical stop will be raised. A critical stop will cause services to stop running.
