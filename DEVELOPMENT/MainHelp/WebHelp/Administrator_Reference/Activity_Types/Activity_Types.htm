Grooper 2.70 Documentation

Activity

Activities define work to be performed on objects in a batch, and define settings which control how the work is performed. Activities fall into two general categories: Attended Activities, which are performed by a human operator in a Grooper client application, and Unattended Activities, which are performed by code.

List of Activity Types

Name Description
Attended Activities
  Scan Scan is an attended activity which provides a user interface optimized for creating batches and scanning/importing pages. Scan is typically the first step in a Batch Process. The user interface for the Scan activity includes the Scan Viewer Batch View control.
Export
  Document Export Exports document content using an Export Provider.
Human Review
  Classification Review Provides a user interface for reviewing the structure of a batch after Classification and/or Separation. The user interface for the Classification Review activity includes the Batch Viewer Batch View control.
  Data Review Provides a user interface for entering data or validating the results of Extraction. The user interface for the Data Review activity includes the Index Navigator Batch View control.
  Image Review Image Review is an attended activity which provides a user interface optimized for quickly reviewing and/or correcting the results of Classification or Separation. The user interface for the Image Review activity includes the Batch Viewer and Thumbnail Viewer Batch View controls.
  Review Provides a configurable user interface for human review of documents or pages in a batch. The Attended Client provides the user interface for this Activity. This client will display one or more Batch View controls as indicated by the Activity's configuration.
Microfilm Processing
  Analyze Strips Category: Microfilm Processing. Performs a horizontal projection profile on each page of a Batch (where each Batch Page is a strip of a microfilm reel) and stores the result as text files on the Batch Page objects. This Activity is intended to be a precursor to the Detect Frames Activity that will use the horizontal profile data to do further work. Each page image is required to be 8 bit grayscale.
  Consolidate Pages Category: Microfilm Processing. A prerequisite for running this Activity is the Detect Pages Activity. After running the Detect Pages Activity, each Frame Folder contains either one or two Batch Pages that were contained within the frame. The Consolidate Pages Activity consolidates all of the Frame Folder Pages as children of the Batch root and deletes the Frame Folders.
  Detect Frames Category: Microfilm Processing. Analyzes horizontal projection profile data stored on a set of microfilm strips to detect microfilm frames and creates one batch folder per frame. A prerequite for running this Activity is the Analyze Strips Activity. After running the Analyze Strips Activity, each BatchPage object contains horizontal projection profile data for the page (which represents a strip of a microfilm reel). The Detect Frames Activity analyzes the horizontal projection profile data for the entire reel to detect each frame. A batch folder will be created for each frame, and the frame image will be stored as the primary image file on the batch folder. Finally, the original page images for the Batch will be deleted. The end result of the Activity is a set of Batch Frame folders.
  Detect Pages Category: Microfilm Processing. For each Frame Folder, this Activity detects and extracts Page Images from the Frame Image and creates child batch page objects under the Frame Folder. A prerequite for running this Activity is the Detect Frames Activity. After running the Detect Frames Activity, each Frame Folder contains a frame image that contains one or two pages. The Detect Pages Activity analyzes the frame image to detect the pages within it.
Miscellaneous
  CMIS Batch Transfer Transfers a Batch to another Grooper repository, optionally launching the batch into a Batch Process in the target repository. This activity is designed to facilitate distributed applications, providing a secure mechanism for moving batches between Grooper repositories over web protocols. To use this activity, the following prerequisites must be met:
  • An instance of Grooper CMIS Server must be installed on the target repository.
  • A CMIS Connection must be created in the source repository, which connects to the Grooper CMIS Server discussed above.
  • The CMIS Repository representing the target repository must be imported in the source repository.
  Deduplicate Detects duplicate documents.
  Dispose Batch Dispose Batch is an unattended activity designed to dispose of Batches when they have completed processing through a Batch Process. This activity can delete Batches, move them to archival, or migrate their contents between Grooper File Stores. Please note that if deleting or archiving the batch, this action must be the last step in the Batch Process.
  Launch Process Launches an external process, optionally waiting for that process to terminate.
  Spawn Batch Creates a new batch using a filtered set of level 1 folders from the current batch.

Copies one or more level 1 folder objects from the batch to a new batch, optionally launching the new batch into a Batch Process. If no items in the batch match the specified criteria, no batch is created.

  Train Lexicon Executes an extractor and populates a Lexicon with the results.
Object Application/Execution
  Execute Command Executes a command defined for a Batch Folder object or Batch Page object.
  Image Processing Applies an IP Profile to pages in a Batch. IP Profiles apply a list of image processing commands to pages in a Batch, and are used for image cleanup and/or image feature detection.
  Redaction Applies a Redaction Profile to pages in a batch. Redaction performs permanent image-based redaction of information based on settings defined in a Redaction Profile.
  Separation Applies a Separation Profile to pages in a Batch. This activity will apply the specified Separation Profile to all pages which are immediate children of the folder being processed. To separate loose pages in the root of a Batch, run this activity at the batch level. To separate loose pages inside a subfolder, run the activity at the level of the folders which contain loose pages.
OCR Processing
  Correct OCR Corrects OCR results for a document, or for a data element on a document. Performs word splitting and OCR spell correction.
  Full Text OCR Applies an OCR Profile to pages in a Batch. This activity can only run at page scope. The selected OCR Profile will be executed against the primary image for each page, and the OCR results will be stored to the page object.
Rendering
  Render Document Renders the native file version stored on a Batch Folder to PDF.

Rendering allows electronic documents of any type to be displayed in Grooper, and allows their textual content to be accessed by Grooper activities such as Classification and Extraction. Conversion to PDF is achieved by automating the native application to print the document to a Grooper- installed printer called the 'Grooper Render Printer', which generates a PDF file from the print stream. This activity can render any electronic document format, as long as the following criteria are met:

  • An instance of Grooper Render Printer has been installed on the machine(s) where the activity will run, and has been set as the default printer for the user account under which the activity will run.
  • The native application for the file type has been installed on the machine(s) where the activity will run.
  • The native application provides a shell print verb (i.e. when the file is right-clicked in Windows Explorer, a 'Print' option is present).

Only a single instance of this activity may run on the same machine at the same time. If the folder does not have a native version, this activity does nothing.

Unattended Activities
  A2iA CheckReader Performs check recognition using A2iA CheckReader. Runs with folder scope only. The first page in the folder to be processed must be the front image of a check. Once A2iA recognition has run, the resulting values are stored to the fields indicated in the Field Mappings property. A2iA CheckReader can extract the following data elements:
  • Amount - The dollar amount of the check.
  • Check Date - The date of the check.
  • Check Number - The check number.
  • Code Line - The MICR line of the check.
  • Payee - The name and address of the payee.
  • Payor - The name and address of the payor.

A2iA CheckReader requires special licensing. To purchase licensing for A2iA CheckReader, please contact your Grooper sales representative.

  Checkify Performs custom processing on check and supporting document images for Check21 Batches Finds check image(s) in the batch, moves them to the front of the batch and ensures that each check front image has a back image. At the end of the process, two subfolders will be created, one containing the check images and one for supporting documents. If no check image is found in the batch, then an exception will be thrown.
  Classify Folders Runs classification on Batch Folders. Attempts to automatically assign a Content Type to each Batch Folder. The activity is configured with a Content Model, which defines the settings to be used for classification, and the set of possible content types are the descendants of the Content Model at the specified Classification Level.

Classification is performed as follows, for each folder in scope:

  1. Load the content for the folder. The content for a folder includes all descendant pages of the folder, unless the Page Scope property of the Content Model is set to FirstPageOnly. In this case, only content for the first page of each folder is loaded.
  2. Compare the content to the training for each possible content type.
  3. Generate a list of content types in descending order by similarity, so that the highest-scoring content types are at the top of the list.
  4. If the top result has a similarity below the Minimum Similarity specified on the Content Model, classification fails. In such cases, if a Default Content Type has been specified on the Content Model, then the folder will be assigned this type. Otherwise the folder will be left unclassified.
  5. If the top result and the 2nd result have a similarity difference less than the Minimum Similarity specified on the Content Model, then classification fails. This helps identify and flag documents which are similar to more than one known document type.
  6. Otherwise, the folder is classified with the content type represented by the top result.
  Classify Pages Runs classification on Batch Pages. This activity generates a list of Page Type candidates and stores it to the page object. This activity must run before Separation under the following circumstances:
  • If the configured Separation Profile is using the 'ESP Auto Separation' method.
  • If the configured Separation Profile is using the 'Events' method, and 'Content Type' events are included.
NOTE: The Full Text OCR activity must execute before Page Classification if the specified Content Model is using lexical classification.
  Content Action Performs an action on the content associated with Batch Folder objects. In the scope of this activity, "content" refers to the sequence of pages assocated with a Batch Folder. For example, this activity can be used to split multipage content into individual page objects, or to merge child page objects into a single multipage file stored on the folder.
  Database Export Exports metadata from a Batch Folder object to a Database Table. Allows Data Fields from multiple levels of a Data Model to be mapped to columns in a Database Table.
  Detect Language Detects the dominant language of each Batch Page of the Batch and marks the page with the appropriate language code identifier.
  Extraction Extraction is an unattended activity which performs data extraction on folders in a Batch. This activity can only run with Folder scope. The folders to be extracted must be classified, and the assigned content type must have a Data Model.
  Link Action Performs an action on the Content Link associated with a Batch Folder object.
  MIME Type Action Performs an action on the MIME Type Handler associated with a Batch Folder object.
  PDF Text Extract Performs text extraction from PDF files.
  X9.37 Merge Creates a X9.37 Deposit file from a batch of check payment information. Runs at Batch or Folder level Scope. In either case, all child nodes of the basenode are looked at to see if they match the configured ' document type and if they do they will be processed. Each child folder is expected to contain at least one batchpage representing the front image of the check. It may also contain a batch page representing the back image of the check. If the batch folder does not contain the back image of the check then a dummy back image will be automatically created from the front image.
