Full Text OCR
Applies an OCR Profile to pages in a Batch. This activity can only run at page scope. The selected OCR Profile will be executed against the primary image for each page, and the OCR results will be stored to the page object.

Inherits from: Character Extract Activity

Properties

Property Name Description
General
Ocr Profile
Type: OCR Profile
The OCR Profile which should be applied to each page. (See OCR Profile)
Rotate Pages
Type: Boolean, Default: True
If enabled, and the selected OCR engine supports orientation detection, pages will be permanently rotated to reflect the orientation detected.
Page Scope
Type: PageScope, Default: AllPages
Indicates which pages of each folder will be processed. Can be one of the following values:
  • AllPages
  • FirstNPages
Page Count
Type: Int32, Default: 1
When Page Scope is set to FirstNPages, this property specifies how many pages within each folder should be OCR'd. In some cases, Classification and/or Extraction may not require full text OCR of every page in the document. For example, if documents can be confidently classified using only the content from page 1, then it might be wasteful to perform full text OCR on every page.
Minimum Confidence
Type: Double, Default: 0.5
Indicates the minimum OCR confidence for a page. Any page which has an average OCR confidence below this value will be flagged.
Error Handling
Error Disposition
Type: IssueDisposition, Default: Flag, Log
Determines what happens when an error occurs processing an activity. A combination of the following flags:
  • None: The issue will be ignored, and the item will complete successfully.
  • Flag: The associated Batch Folder or Batch Page will be flagged.
  • Log: The issue will be logged to the Grooper log.
  • Stop: The Batch will be set to error.
Maximum Consecutive Errors
Type: Int32, Default: 0
The maximum number of consecutive errors, after which a critical stop will be raised. A critical stop will cause services to stop running.
