Grooper 2.70 Documentation

Scan
Scan is an attended activity which provides a user interface optimized for creating batches and scanning/importing pages. Scan is typically the first step in a Batch Process. The user interface for the Scan activity includes the Scan Viewer Batch View control.

Inherits from: Attended Client

Properties

Property Name Description
General
Batch Scan Control Settings
Type: Batch Scan Control Settings
Settings which define the behavior of the Batch Scan Control. (See Batch Scan Control Settings)
Required Page Level
Type: Int32, Default: 0
Specifies the level at which all pages must exist before a batch can be completed through scan. A value of zero indicates that pages may exist at any level.
Allow Completion with Flagged Pages
Type: Boolean, Default: False
Specifies whether a batch can be completed from Scan while it has flagged pages. Use this setting to force correction of image problems in the scan module.
UI Configuration
Command Options
Defines the enabled state and shortcut keys for individual commands in Attended Client. By default, all commands are enabled. Use this property to hide commands which don't apply to the task being performed.
User Activity Timeout
Type: Int32, Default: 0, Range: 0 - 480
Specifies the amount of time, in minutes, that a user can be inactive before an automatic logout occurs. This is intended to maintain user statistic integrity that could be damaged by users leaving the application open and unattended. A value of 0 will disable this feature.
