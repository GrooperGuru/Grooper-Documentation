Detect Pages

Category: Microfilm Processing. For each Frame Folder, this Activity detects and extracts Page Images from the Frame Image and creates child batch page objects under the Frame Folder. A prerequite for running this Activity is the Detect Frames Activity. After running the Detect Frames Activity, each Frame Folder contains a frame image that contains one or two pages. The Detect Pages Activity analyzes the frame image to detect the pages within it using two different methods. The first method uses projection profile analysis to detect the rough boundaries for each page. The second method uses computer vision techniques on the frame image to more accurately detect the page boundaries. The first method is less accurate than the second method but is more robust. Both methods are used to calculate the page rectangles on the frame image and the rectangles detected by each method are compared. If page rectangles are detected by the second method and these rectangles cross validate with the rectangles detected by the first method, then the second method is used to accurately extract the page rectangle(s) from the frame image. Otherwise, the first method is used to extract the page rectangle(s) from the frame image and image cleanup is performed on the extracted page images to perform black border removal and image deskew.

Properties

Property Name Description
General
Page Detection Ip Profile An Ip Profile to perform page detection on each Frame Image. This Ip Profile is expected to contain the Edge Detect Ip Command. Edge Detection is used to detect the lines delimiting each page. (See IP Profile)
Image Cleanup Ip Profile An Ip Profile to perform image cleanup on each page extracted using the Projection Profile Method. This Ip Profile is expected to contain commands such as Black Border Removal and Deskew. Note: this is used only when the pages are extracted using the Projection Profile method. There is no need to do this image cleanup when extracting using the more accurate Edge Detection method. (See IP Profile)

Error Handling
Error Disposition Determines what happens when an error occurs processing an activity. A combination of the following flags:
  • None: The issue will be ignored, and the item will complete successfully.
  • Flag: The associated Batch Folder or Batch Page will be flagged.
  • Log: The issue will be logged to the Grooper log.
  • Stop: The Batch will be set to error.
Maximum Consecutive Errors The maximum number of consecutive errors, after which a critical stop will be raised. A critical stop will cause services to stop running.
