Table Extract Method

Defines settings and logic for a Data Section extraction method.

List of Table Extract Method Types

Name Description
Header-Value Detects the layout of a table automatically by analyzing results from value and header extractors defined on Data Column objects.

This table extraction method is designed for structured tables where the following conditions are true:

  • Information is presented in horizontal rows and vertical columns.
  • A header row is present, where text labels indicate the meaning of the data in each column.
  • Table rows occupy one or more full lines of text within the content flow, appearing one after another.
  • The values for all "primary" columns are positioned on the same line of text.

As an example of usage, consider the tabular data provided on invoices. While the location, order, and number of columns in the table may vary from one invoice document to the next, all invoice line items will at minimum specify a quantity, unit price, and line total. For the purposes of Header-Value table extraction, these are considered the 'primary' columns.

Primary columns are a set of required columns which are used to detect table rows. For each primary column, a Header Extractor and a Value Extractor should be defined. A Footer Extractor may also be defined to provide an absolute lower boundary for the last row in the table.

The general operation of this command is as follows:
  • Header detection uses the header extractors defined on primary columns to locate the table header.
  • If defined, the Footer Extractor is executed to locate the table footer.
  • Row detection uses the value extractors defined on primary columns to locate "primary lines".
  • Primary lines are text lines occuring after the header and before the footer which contain at least one value for each primary column.
  • If the primary line contains only one possible value for the column, the value is accepted without checkng the header.
  • If the primary line contains multiple possible values for the column, header information is used to resolve the conflict.
  • Each primary line is then adjusted to consume additional text lines associated with the table row.
  • The value extractors for secondary columns are executed against the content of the table row..
Infer Grid Extracts tables which have both row and column headers, by infering a grid from the header positions.

For example, consider a Data Table object with 4 Data Columns: Month, Revenue, Expenses, and Profit. To extract the data shown below into this table, the following configuration steps are required:

  • Configure the X Axis Extractor to match the column headers.
  • Configure the Y Axis Extractor to match the row headers
  • Specify the 'Month' Data Column as the Header Column.

Additional notes on Infer Grid table extraction:

  • This table extraction method does not support cases where table data spans multiple pages.
  • See notes on the X Axis Extractor and Y Axis Extractor properties for information on how to properly construct header extractors.
  • If multiple instances of the table are found on the document, all rows from all instances will be combined into a single output table.
Row Match Uses an extractor to match each row.
