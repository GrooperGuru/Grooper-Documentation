Infer Grid

Extracts tables which have both row and column headers, by infering a grid from the header positions.

For example, consider a Data Table object with 4 Data Columns: Month, Revenue, Expenses, and Profit. To extract the data shown below into this table, the following configuration steps are required:

  • Configure the X Axis Extractor to match the column headers.
  • Configure the Y Axis Extractor to match the row headers
  • Specify the 'Month' Data Column as the Header Column.

Input Table

RevenueExpensesProfit
January10,00011,00012,000
February6,0006,0006,000
March4,0005,0006,000

Output Data

MonthRevenueExpensesProfit
January10,00011,00012,000
February6,0006,0006,000
March4,0005,0006,000
If the data is transposed on the document, as shown below, enabling the Transpose option will correctly map data to the output table.

Input Table

JanuaryFebruaryMarch
Revenue10,0006,0004,000
Expenses11,0006,0005,000
Profit12,0006,0006,000

Output Data

MonthRevenueExpensesProfit
January10,00011,00012,000
February6,0006,0006,000
March4,0005,0006,000

Additional notes on Infer Grid table extraction:

  • This table extraction method does not support cases where table data spans multiple pages.
  • See notes on the X Axis Extractor and Y Axis Extractor properties for information on how to properly construct header extractors.
  • If multiple instances of the table are found on the document, all rows from all instances will be combined into a single output table.

Properties

Property Name Description
General
X Axis Extractor
Type: Embedded Extractor
The Extractor to be used to identify column headers. The X Axis Extractor should match the entire header row, and output one child instance for each column. When Transpose is set to false, it is important that the child instance names match the names of the Data Column. (See Embedded Extractor)
Y Axis Extractor
Type: Embedded Extractor
The Extractor to be used to identify row headers. (See Embedded Extractor)
Transpose
Type: Boolean, Default: False
If true, rows from the source table will become columns in the output table.
Detect Lines
Type: Boolean, Default: True
Enables or disables line detection during table extraction. When line detection is enabled, horizontal and vertical lines on the document will be used to determine the boundaries of rows and columns. Disabling line detection can provide a performance increase, but could reduce accuracy on certain table formats.
Header Column
Type: Data Column
Specifies a column to receive the row header values. (See Data Column)
