Grooper 2.70 Documentation

File Store
A File Store defines a storage location where Grooper can store content files associated with nodes in the Grooper tree structure. Files are stored for a variety of purposes in Grooper, including but not limited to:
  • Scanned and imported images are stored as files on Batch Page objects.
  • PDF Versions of imported PDF documents are stored as files on Batch Folder objects.
  • OCR results are stored as files on Batch Page objects.
  • Script source code and compiled script binaries are stored as files on Grooper objects which support scripting.

When new files are added to Grooper, they are always written to the File Store indicated by the Active File Store property of the Grooper Root object.

Inherits from: Grooper Node

Properties

Property Name Description
General
Storage Path
Type: String
The file system path on which this file store will be hosted. NOTE: Network paths should be specified using UNC paths. Using mapped drives will cause system errors when service processes attempt to access the file store. This setting can only be changed when the file store is empty.
Description
Type: String
Generic property allowing an administrator to document the purpose of this Grooper Node.
File Store Information
Total File Count
Type: String
The total number of files in the file store.
Disk Free Space
Type: String
The amount of free disk space.
