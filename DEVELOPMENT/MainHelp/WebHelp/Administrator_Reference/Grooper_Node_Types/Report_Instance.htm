Report Instance

Represents a named configuration of a Report Definition. Report Instances are used by applications such as Grooper Dashboard and Grooper Kiosk to display reports. For example, consider a Report Definition named 'Scan Volume' which has Start Date and End Date properties. A Report Instance with the Start Date set to '01/01/2017' and the End Date set to '12/31/2017' could be saved and named 'Scan Volume - 2017'. The 'Scan Volume - 2017' Report Instance could then be chosen when configuring which Report Instances to display in Grooper Kiosk.

There are several predefined Report Definitions which ship with Grooper, but custom reports can be created in an Object Library Script by users with the appropriate skill sets using Microsoft Visual Studio and the Visual Studio Report Designer..
