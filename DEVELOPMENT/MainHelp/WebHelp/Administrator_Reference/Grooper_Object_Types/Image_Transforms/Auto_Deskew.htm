Auto Deskew
Detects the skew angle of an image, and optionally rotates the image to compensate. This command works by rotating a low-resolution copy of the image through a sequence of angles, and analyzing the image at each orientation to determine how straight it is.

The angles to be tested are defined by Maximum Angle and Angle Precision. The algorithm will iterate from -Maximum Angle to +Maximum Angle in increments of Angle Precision, testing the straightness of the image at each angle. For example, a 5° maximum angle with 0.25° angle precision will test 41 different angles: [-5.00°, -4.75°, -4.50°, ..., 0.00°, ..., 4.50°, 4.75°, 5.00°]. Therefore, a higher maximum angle or lower angle precision will result in more distinct angles being tested, reducing the speed of the deskew operation, but increasing its angular detection range or accuracy.



Inherits from: IP Command

Represents a trade-off between speed and accuracy during skew detection. Lower resolutions increase speed, at the risk of reduced accuracy. Most standard document images can be deskewed accurately at 25 DPI or lower.
Maximum Angle
Type: Double, Default: 5°, Range: 0° - 45°
The maximum skew angle to detect. Increasing the maximum skew angle increases processing time, because it increases the total number of distinct angles to be tested. Skew angles outside of this range will not be detected.
Angle Precision
Type: Double, Default: 0.25°, Range: 0° - 1°
The precision at which skew angles are detected. Represents the angle increment, in degrees, for each iteration of skew angle detection. The default is 0.25°, meaning that images will be deskewed to a precision of 1/4°. Decreasing the angle precision increases processing time, as more distinct angles will be tested.
Interpolation Mode
Type: RotateInterpolationMode, Default: Linear
The interpolation mode to use during skew detection. Can be one of the following values:
  • Linear: Linear interpolation is slower but more accurate than the Nearest Neighbor interpolation. It is faster but less accurate than Cubic Interpolation. Linear interpolation sets the destination pixel based on analysis of the surrounding 4 pixels in the source image.
  • Cubic: The most accurate, but slowest interpolation method. Sets destination pixels based on a 16-pixel neighborhood in the source image.
  • NearestNeighbor: This is the fastest and least accurate interpolation mode. This simply sets the destination pixel to the value of the closest source pixel.
This is the interpolation method internally during skew detection. Affects the speed and accuracy of skew angle detection, but does not impact the quality of the final rotated image.
Border Size
Type: Double, Default: 0, Range: 0 - 2
The size, in inches, of a border around the edge of the image which will be ignored during skew detection. In some cases, edge artifacts may interfere with deskewing.
Skew Correction
Detect Only
Type: Boolean, Default: False
If true, the skew angle will be detected, but the image will not be rotated to compensate.
Interpolation Mode
Type: RotateInterpolationMode, Default: Linear
The interpolation mode to use when rotating the final image to correct a detected skew angle. Can be one of the following values:
  • Linear: Linear interpolation is slower but more accurate than the Nearest Neighbor interpolation. It is faster but less accurate than Cubic Interpolation. Linear interpolation sets the destination pixel based on analysis of the surrounding 4 pixels in the source image.
  • Cubic: The most accurate, but slowest interpolation method. Sets destination pixels based on a 16-pixel neighborhood in the source image.
  • NearestNeighbor: This is the fastest and least accurate interpolation mode. This simply sets the destination pixel to the value of the closest source pixel.
Controls the quality of the final deskewed image.
Background Color
The color used to fill in edges exposed by rotation. If no background color is specified, the dominant background color of the image will be used.
Resize
Type: Boolean, Default: False
If true, the image will be enlarged so that the entire rotated image fits into the destination image.
