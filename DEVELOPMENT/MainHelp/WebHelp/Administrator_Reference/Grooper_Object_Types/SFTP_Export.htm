Grooper 2.70 Documentation

SFTP Export
Provides export connectivity over the SFTP protocol.

Inherits from: [File Export]

Properties

Property Name Description
Server Connection
Host Name
Type: [String]
The Url of a Base FTP directory containing one or more other FTP directories and/or files.
Port No
Type: [Int32], Default: 22
The port number to use when communicating with the server. Port 22 is the standard port number for SFTP.
User Name
Type: [String]
The user name to be used for accessing the server.
Password
Type: [String]
The password to be used for accessing the server.
Connect Timeout
Type: [Int32], Default: 30, Range: 1 - 6000
The server connection timeout in seconds.
Content Export
Base Export Folder
Type: [String]
The server folder to which content will be exported. To write files to the root directory, leave this property blank.
General
File Export Settings
Type: [Folder Content Generator], Default: Native
Defines the settings which control the format of the output file.
Update Link
Type: [Boolean], Default: True
If true, the [Content Link] on the document will be updated to point at the exported content.
File and Directory Naming
Content Type
Type: [Content Type]
The Grooper content type to be used for mapping field values to folder and file names. If only a single content type will be encountered during export, select that content type directly. If multiple content types are to be exported, the content type selected here must be a parent content type, from which all content types inherit. If a folder is encountered with a content type which does not equal or derive from the type specified here, the folder will be ignored. (See [Content Type])
Subfolder Levels
Type: [Int32], Default: 0
The number of sub folder levels to be created. Mappings must be created on the Folder Mappings tab for each level specified, and a Content Type with a Data Model must be specified.
Folder Mappings
Defines mappings between [Data Fields] in a Grooper [Data Model] and the file / folder names in the export path.
Metadata Export
Metadata Format
Type: [MetaDataFormat], Default: None
The format in which metadata will be exported. Can be one of the following values:
  • None: A metadata file will not be exported.
  • XML: Metadata will be exported in XML format.
  • JSON: Metadata will be exported in JSON format.
  • TextFile: Metadata will be exported as rows in a flat text file.
Metadata is exported as a companion file having the same name as the content file, but with an extension reflecting the selected format.
Delimiter Character
The character to be used as a delimiter when a delimited text file is being created.
Use Quotes
Type: [Boolean], Default: False
If set to true, field values will be enclosed in quotes when a delimited text file is being created.
Include File Path
Type: [Boolean], Default: False
If set to true, the path to the exported content file will be included as the last field value when a delimited text file is being created.
Include Header
Type: [Boolean], Default: False
If set to true, field names will be included as the first row when a delimited text file is being created.
