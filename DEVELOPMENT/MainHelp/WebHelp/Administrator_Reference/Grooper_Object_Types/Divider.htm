Divider

Identifies Section Instances by splitting the input content at each occurence of a specific text pattern. This method is useful for processing content with repeating sections of information which occur sequentially in the text flow of the document.

For example, the document below could be divided into employee records by splitting the content at each occurence of the text pattern '\r\nEMPLOYEE:'.


ACTIVE EMPLOYEE REPORT

EMPLOYEE: Doe, Jane SSN: 000-00-0000 PHONE: (000) 000-0000
ADDRESS: 1234 S Main CITY: Anytown STATE: AA ZIP: 00000
DOB: 00/00/0000 EMAIL: janedoe@acme.com DEPARTMENT: IT

EMPLOYEE: Doe, John SSN: 000-00-0000 PHONE: (000) 000-0000
ADDRESS: 2233 N Elm CITY: Anytown STATE: AA ZIP: 00000
DOB: 00/00/0000 EMAIL: johndoe@acme.com DEPARTMENT: AP

EMPLOYEE: Smith, Kim SSN: 000-00-0000 PHONE: (000) 000-0000
ADDRESS: 8888 W 24th CITY: Anytown STATE: AA ZIP: 00000
DOB: 00/00/0000 EMAIL: kimsmith@acme.com DEPARTMENT: Admin



Inherits from: Section Extract Method

Properties

Property Name Description
General
Divider Extractor
Type: Embedded Extractor
The extractor to be used to identify section dividers. The input will be split at each extracted instance according to the Split Position property. (See Embedded Extractor)
Split Position
Type: SplitPositionEnum, Default: Begin
Indicates how the input is to be split relative to the instances returned by the Divider Extractor. Can be one of the following values:
  • Begin: Specifies that the input should be split at the beginning of each match.
  • End: Specifies that the input should be split at the end of each match.
  • Around: Specifies that the input should be split around each match. The number of instances returned will always be the number of extractor hits + 1.
  • Between: Specifies that the input should be split so that the output contains the data between each instance. The number of instances returned will always be the number of extractor hits - 1. As such, at least two extractor hits are required for any result instances to be returned.
