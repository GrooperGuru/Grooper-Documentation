Grooper 2.70 Documentation

Detect Pages
Category: Microfilm Processing. For each Frame Folder, this Activity detects and extracts Page Images from the Frame Image and creates child batch page objects under the Frame Folder. A prerequite for running this Activity is the [Detect Frames] Activity. After running the [Detect Frames] Activity, each Frame Folder contains a frame image that contains one or two pages. The [Detect Pages] Activity analyzes the frame image to detect the pages within it.

Inherits from: [Unattended Activity]

Properties

Property Name Description
General
Image Cleanup Ip Profile
Type: [IP Profile]
An [Ip Profile] to perform image cleanup on each page extracted using the Projection Profile Method. This [Ip Profile] is expected to contain commands such as Black Border Removal and Deskew. (See [IP Profile])
Microfilm Recording Method
Type: [RecordingMethod], Default: Either
Describes how page images have been recorded on each frame of the microfilm reel. Can be one of the following values:
  • Simplex: Each frame contains a single image.
  • Duplex: Each frame contains two images side by side.
  • Either: Each frame contains either one or two images.
This property is used to validate that each detected frame contains the expected number of page images.

Error Handling
Error Disposition
Type: [IssueDisposition], Default: Flag, Log
Determines what happens when an error occurs processing an activity. A combination of the following flags:
  • None: The issue will be ignored, and the item will complete successfully.
  • Flag: The associated [Batch Folder] or [Batch Page] will be flagged.
  • Log: The issue will be logged to the Grooper log.
  • Stop: The [Batch] will be set to error.
Maximum Consecutive Errors
Type: [Int32], Default: 0
The maximum number of consecutive errors, after which a critical stop will be raised. A critical stop will cause services to stop running.
