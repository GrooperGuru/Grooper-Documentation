Field Class
A Field Class is a trainable binary classifier, used for locating specific instances of a data pattern on a document. Field Classes are useful when a data value appears more than one time on a document, and it is necessary to examine the context of each data value. For example, a Field Class could be used to determine which date on an invoice document represents the Invoice Date, or which paragraph on a legal contract represents a Non-Compete Clause.

Internally, a Field Class is classifier with two classes: Positive and Negative. When training is performed, the training instance is trained as a positive instance, and all other instances are trained as negative instances. For example, if the Invoice Date field is trained, then the Ship Date and Due Date will be trained as negative instances.

The instances output from a Field Class will have a confidence value which reflects how closely the context matches the training data.

Properties

Property Name Description
General
Value Extractor
Type: [Embedded Extractor]
The extractor to use for data values. (See [Embedded Extractor])
Feature Extractor
Type: [Embedded Extractor]
The extractor to use for context features. When 'Self' scope is in use, this extractor will execute against the values returned by the Value Extractor. In all other scope modes, this extractor executes against the full document content, and is then trimmed down to meet the scope criteria. For example, with zonal scope only features which have a centerpoint inside a context zone will be included. (See [Embedded Extractor])
Description
Type: [String]
Generic property allowing an administrator to document the purpose of this Grooper Node.
Context Scope Options
Context Scope
Type: [ContextScopeEnum], Default: Zonal
Determines the scope of context feature extraction. Can be one of the following values:
  • Zonal: Context features will be extracted from one or more zones, specified relative to the data value.
  • Flow: Context will include a limited number matching features before and/or after the data value in the text flow.
  • Self: Context will include all matching features which occur inside of or overlap with the data value.
  • Nearest: Context will include a limited number of features which are closest to the data value.
Max Instances
Type: [Int32], Default: 0
The maximum number of nearby features to include.
Direction Filter
Type: [CompassDirection], Default: 0
Limits the context features included based on their geometric position relative to the data value. A combination of the following flags:
  • North: Above the data value. Values on a previous page are also considered north.
  • South: Below the data value. Values on a subsequent page are also considered south.
  • West: Left of the data value.
  • East: Right of the data value.
  • NorthWest: Above and left of the data value.
  • NorthEast: Above and right of the data value.
  • SouthWest: Below and left of the data value.
  • SouthEast: Below and right of the data value.
  • All: All Directions
Direction Filter
Type: [FlowDirection], Default: 0
Limits the context features included based on their position in the text flow relative to the data value. Can be one of the following values:
  • Before: Occurs before the data value in the text flow.
  • After: Occurs after the data value in the text flow.
  • Either: Occurs before or after the data value in the text flow.
Context Zones
Defines zones relative to a data value from which context features will be extracted. Context zones are specified as relative coordinates around a located value. The unit of measure is inches. For example, a value of -2 for the left side of a context zone will place the left edge of the context zone 2" left of the value.
Exclusion Extractor
Type: [Embedded Extractor]
An optional extractor which allows data values to be excluded as potential results when specific features appear in the context zones. If this extractor produces any matches within a context zone, the data value will be excluded as a potential result. (See [Embedded Extractor])
Maximum Distance
Type: [Double], Default: 0
When nearest context mode is in use, specifies the maximum distance, in inches, between a context feature and the data value. A value of 0 indicates no maximum distance.
GeoTag Features
Type: [Boolean], Default: False
If enabled, features will be tagged with suffixes which indicate their position relative to the data value. The definition of 'position' depends on the Context Scope:
  • When Flow scope is in use, a (b) or (a) suffix will indicate whether the feature occurs before or after the data value in the text flow.
  • When Zonal or Nearest scope is in use, a (n), (s), (e), (w), (ne), (nw), (se), or (sw) suffix will indicate the geometric position of the feature relative to the data. A feature occuring on a previous page will be considered (n) of the data value, and feature occuring on a subsequent page will be considered (s).
  • This option is not available when Self scope is in use, as all context features are located inside the data value.
Classifier Tuning
Minimum Feature Count
Type: [Int32], Default: 0
Indicates the minimum number of features required for a candidate instance.
Training Threshold
Type: [Double], Default: 0%, Range: 0% - 100%
Determines the threshold of similarity which triggers negative instance training.
Use Class Frequency
Type: [Boolean], Default: False
If true, class frequency will be considered by the classifier.
Output Options
Minimum Confidence
Type: [Double], Default: 0%, Range: 0% - 100%
The minimum classification confidence value for an instance to be output. A value of zero will include every instance produced by the Value Extractor.
Collation Method
Type: [CollationType], Default: Individual
Controls how results from individual extractors are processed into a final output result. Can be one of the following values:
  • Individual: All qualifying results will be returned individually. Use this setting to return a list of multiple candidates.
  • Combine: All qualifying results will be combined into a single instance.
  • Boolean: If any qualifying results are found, returns "true". Otherwise, returns "false".
See [Collation Examples] for examples of each different collation method.
Separator
Type: [String]
The separator to be used between each combined instance.
Instance Ranking
Type: [RankingMethod], Default: Confidence
Controls the output order of the results returned. Can be one of the following values:
  • Order: Results are returned in the order in which they appear in the content flow.
  • ReverseOrder: Results are returned in reverse of the order in which they appear in the content flow.
  • Frequency: Results are returned ordered by the number of occurrences of each value in the document. Value which occur more frequently will be sorted to the top.  For example, a mortgage file may contain many different SSN values, but the most frequently occurring SSN value will be that of the borrower.  In such a case, a Pattern-based extraction with a Instance Ranking based on Frequency will produce the correct result for the Borrower SSN field in most cases.
  • Confidence: DO NOT USE.  This ranking method is used internally by Grooper to rank instances extracted by a Field Class during context-based extraction, after the confidence value for each instance has been calculated.
  • OcrConfidence: Results are returned order by the average OCR confidence of all characters which make up the extracted value.
  • ExtractionOrder: Results are returned in the order of the extractor which produced each match. Can be used to prioritize the results from one extractor those of over another.
  • ValueLengthAscending: Results are returned in the order of their value lengths, in ascending order.
  • ValueLengthDescending: Returns are returned in the order of their value lengths, in descending order.
