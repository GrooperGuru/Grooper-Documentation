Registration Zone

Registration zones are used to align [Field Zones] and [OMR Zones] prior to zonal extraction. When documents are scanned, pages shift slightly, causing alignment issues with zonal extraction. Registration zones correct this shift by aligning zones relative to other elements on the page.

Registration zones cannot have children. Registration zones can be children of [Field Zones] or [OMR Zones].

Inherits from: [Field Zone]

Properties

Property Name Description
Registration
Scope
Type: [RegistrationZoneScope], Default: Page
The scope of the registration zone. Can be one of the following values:
  • Page: The registration zone on this page will be search for instances of the registration text.
  • Document: The entire document content will be searched for instances of the registration text. Requires full text OCR.
  • PageRegion: Aligns the field to a specific region of the page.
Location
Type: [Rectangle], Default: (0,0):(0,0)
The location of the registration text on the sample page used for setup. This location must be set before the registration zone can be used. To set the location, test the Registration Zone and use the Save Location command.
Registration Pattern
Type: [Data Pattern]
Defines a text value or pattern which matches a label or other text element near the data value. The registration pattern is responsible for matching an element on the page, such as a field label, which will always have the same position relative to the data value. If the registration pattern hits multiple locations, an extraction will be attempted at each of the locations, potentially producing multiple candidates for the field value. (See [Data Pattern])
Alignment
Type: [RegistrationAlignment], Default: TopLeft
Indicates which corner of the registration value is used as the registration point. Can be one of the following values:
  • TopLeft: Registration will be relative to the top left of the registration text.
  • TopRight: Registration will be relative to the top right of the registration text.
  • BottomLeft: Registration will be relative to the bottom left of the registration text.
  • BottomRight: Registration will be relative to the bottom right of the registration text.
  • Center: Registration will be relative to the center of the registration text.
In some cases, it can be important which corner of a text element is used as the reference point. This is particularly true when regular expression syntax is used in the registration text, and the resulting registration text may have a variable length.
Use At Page Level
Type: [Boolean], Default: False
If enabled, this registration zone will be used for all zones on the page which do not have individually-defined registration zones.
Extraction
Extraction Type
Type: [ZonalExtractionType], Default: None
The type of data extraction to be used with this zone. Can be one of the following values:
  • None: No zonal extraction will be performed. This zone simply establishes the zoom area when the user tabs into the field.
  • Barcode: The zone will use barcode recognition to read a barcode value.
  • OCR: The zone will use OCR to read a text value.
  • FullText: The zone will be re-extracted from existing full text OCR, if it exists.
OCR Profile
Type: [OCR Profile]
The [OCR Profile] to be used for zonal data extraction. (See [OCR Profile])
Barcode Detection Settings
Type: [Barcode Detection]
The settings to be used for barcode recognition. (See [Barcode Detection])
Apply Field Extractor
Type: [Boolean], Default: False
Controls whether the extractor for the field will be executed on the OCR results obtained from the zone.
Shape Name
Type: [String]
The name of a Shape to be used as the region for this zone.
Zone Location
Page No
Type: [Int32], Default: 0
The page number associated with this zone. A negative value indicates on offset starting at the end of the document. For example, -1 indicates the last page of the document -2 indicates the 2nd to last page of the document, and etc.
Bounds
Type: [Rectangle], Default: (0,0):(0,0)
The location of the zone.
Image Cleanup
IP Profile
Type: [IP Profile]
The [IP Profile] to run prior to zonal extraction. This IP Profile performs temporary image cleanup prior to extraction. It does not permanently alter the image. (See [IP Profile])
