Grooper 2.70 Documentation

Train Lexicon
Executes an extractor and populates a [Lexicon] with the results.

Inherits from: [Unattended Activity]

Properties

Property Name Description
General
Lexicon
Type: [Lexicon]
The [Lexicon] to be trained. (See [Lexicon])
Extractor
Type: [Embedded Extractor]
Defines an extractor which will be used to define the values available to train the [Lexicon]. (See [Embedded Extractor])
Scope
Type: [TrainingScope], Default: Document
Indicates the scope against which the extractor should be executed. Can be one of the following values:
  • Document: The extractor will be executed against the full document content.
  • DataElement: The extractor will be executed against the content of a [Data Element].
Page Scope
Type: [Int32], Default: 0
Indicates a maximum number of pages from each document to be trained. A value of zero indicates unlimited pages.
Data Element
Type: [Data Element]
When Scope is set to 'DataElement', indicates the DataElement against which the extractor will be run. (See [Data Element])
Error Handling
Error Disposition
Type: [IssueDisposition], Default: Flag, Log
Determines what happens when an error occurs processing an activity. A combination of the following flags:
  • None: The issue will be ignored, and the item will complete successfully.
  • Flag: The associated [Batch Folder] or [Batch Page] will be flagged.
  • Log: The issue will be logged to the Grooper log.
  • Stop: The [Batch] will be set to error.
Maximum Consecutive Errors
Type: [Int32], Default: 0
The maximum number of consecutive errors, after which a critical stop will be raised. A critical stop will cause services to stop running.
