Grooper 2.70 Documentation

Line Detection
Detects lines in an image using a Hough Transform. This command does not produce an end result by itself, but rather is used in other IP commands, such as Sticky Note Detection, to detect line features in images.

Inherits from: IP Command

Properties

Property Name Description
Preprocessing
Binarization Settings
Type: Binarize
The settings to be used for binarizing color and grayscale images. (See Binarize)
General
Maximum Lines
Type: Int32, Default: 8096
The maximum number of lines to detect. Line detection will fail if more lines are detected
Minimum Votes
Type: Int32, Default: 16
The minimum number of votes for a successful line candidate.
Minimum Line Length
Type: String, Default: 1in
The minimum length of a line.
Maximum Line Gap
Type: String, Default: 0.1in
The maximum gap between detected line segments.
Delta
Type: Double, Default: 1
Controls the number of bins for distance from the origin.
Angle Precision
Type: Double, Default: 0.25°
Controls the number of bins for line angle.
Border Size
Type: String
The size of a border around the edge of the page where line detection will be suppressed.
Result Filtering
Line Types
Type: AngleCategory, Default: Horizontal, Vertical, NonSquareAngles
The type of lines to be returned. A combination of the following flags:
  • Horizontal
  • Vertical
  • NonSquareAngles
Square Angle Tolerance
Type: Double, Default: 1°
The tolerance to be used when determining if lines qualify as vertical or horizontal lines.
Merge Duplicate Lines
Type: Boolean, Default: False
If true, then lines that are within a certain distance and angle of each other will be merged into a single line.
Angle Tolerance
Type: Double, Default: 0°
The Maximum angle variation between two lines to be considered a candidate for duplicate line merging.
Distance Tolerance
Type: String, Default: 0.1in
The Maximum distance between two lines to be considered a candidate for duplicate line merging.

The pixel formats supported by this command.
