X9.37 Merge
Creates a X9.37 Deposit file from a batch of check payment information. Runs at Batch or Folder level Scope. In either case, all child nodes of the basenode are looked at to see if they match the configured ' document type and if they do they will be processed. Each child folder is expected to contain at least one batchpage representing the front image of the check. It may also contain a batch page representing the back image of the check. If the batch folder does not contain the back image of the check then a dummy back image will be automatically created from the front image.

Inherits from: [Unattended Activity]

Error Disposition
Type: [IssueDisposition], Default: Flag, Log
Determines what happens when an error occurs processing an activity. A combination of the following flags:
