Classify Folders
Runs classification on [Batch Folders]. Attempts to automatically assign a [Content Type] to each [Batch Folder]. The activity is configured with a [Content Model], which defines the settings to be used for classification, and the set of possible content types are the descendants of the Content Model at the specified Classification Level.

Classification is performed as follows, for each folder in scope:

  1. Load the content for the folder. The content for a folder includes all descendant pages of the folder, unless the Page Scope property of the Content Model is set to FirstPageOnly. In this case, only content for the first page of each folder is loaded.
  2. Compare the content to the training for each possible content type.
  3. Generate a list of content types in descending order by similarity, so that the highest-scoring content types are at the top of the list.
  4. If the top result has a similarity below the Minimum Similarity specified on the Content Model, classification fails. In such cases, if a Default Content Type has been specified on the Content Model, then the folder will be assigned this type. Otherwise the folder will be left unclassified.
  5. If the top result and the 2nd result have a similarity difference less than the Minimum Similarity specified on the Content Model, then classification fails. This helps identify and flag documents which are similar to more than one known document type.
  6. Otherwise, the folder is classified with the content type represented by the top result.


Inherits from: [Classify Object]

Properties

Property Name Description
General
Classification Level
Type: [ClassificationLevel], Default: DocType
The level at which the set of allowed content types exists, specified relative to the base node indicated by Content Model Scope. Can be one of the following values:
  • DocType: Classifies to the [Document Type] level. This setting is only valid when the task is running with Folder scope.
  • Level1: Classifies to level 1 of the [Content Model].
  • Level2: Classifies to level 2 of the [Content Model].
  • Level3: Classifies to level 3 of the [Content Model].
  • Level4: Classifies to level 4 of the [Content Model].
  • Level5: Classifies to level 5 of the [Content Model].
  • Level6: Classifies to level 6 of the [Content Model].
  • Level7: Classifies to level 7 of the [Content Model].
  • Level8: Classifies to level 8 of the [Content Model].
  • PageType: Classifies to the [Page Type] level. This setting is only valid when the task is running with Page scope.
  • FormType
Content Model Scope
Type: [Content Type]
The [Content Model] or [Content Category] containing the set of allowed [content types]. (See [Content Type])
Model Refresh Interval
Type: [Int32], Default: 60, Range: 0 - 2147483647
The interval (in seconds) at which content model information will be refreshed from the repository.
Error Handling
Error Disposition
Type: [IssueDisposition], Default: Flag, Log
Determines what happens when an error occurs processing an activity. A combination of the following flags:
  • None: The issue will be ignored, and the item will complete successfully.
  • Flag: The associated [Batch Folder] or [Batch Page] will be flagged.
  • Log: The issue will be logged to the Grooper log.
  • Stop: The [Batch] will be set to error.
Maximum Consecutive Errors
Type: [Int32], Default: 0
The maximum number of consecutive errors, after which a critical stop will be raised. A critical stop will cause services to stop running.
