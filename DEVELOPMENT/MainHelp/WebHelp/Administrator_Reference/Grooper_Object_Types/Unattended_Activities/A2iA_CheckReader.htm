A2iA CheckReader
Performs check recognition using A2iA CheckReader. Runs with folder scope only. The first page in the folder to be processed must be the front image of a check. Once A2iA recognition has run, the resulting values are stored to the fields indicated in the Field Mappings property. A2iA CheckReader can extract the following data elements:
  • Amount - The dollar amount of the check.
  • Check Date - The date of the check.
  • Check Number - The check number.
  • Code Line - The MICR line of the check.
  • Payee - The name and address of the payee.
  • Payor - The name and address of the payor.

A2iA CheckReader requires special licensing. To purchase licensing for A2iA CheckReader, please contact your Grooper sales representative.



Inherits from: [Unattended Activity]

Path to the A2iA Check Table Definitions Table File. This is a file containing field definitions for a U.S. check and is used to know the locations of each field for OCR/ICR purposes.
Dongle Server Name
Type: [String]
Name or IP Address to the A2iA DualWorld Protection Server. The A2iA DualWorld Protection Server is used for A2iA CheckReader licensing.
Dongle Server Port
Type: [String], Default: 5012
The port A2iA DualWorld Protection Server is running on.
Minimum Confidence Score
Type: [Int32], Default: 250
The minimum confidence score (ranging from 1 to 1000) for which an A2iA extracted field value will be imported into Grooper.

Type: [IP Profile]

Field Mappings
Defines mappings between fields in A2iA CheckReader and [Data Fields] in a Grooper [Data Model].
Error Handling
Error Disposition
Type: [IssueDisposition], Default: Flag, Log
Determines what happens when an error occurs processing an activity. A combination of the following flags:
  • None: The issue will be ignored, and the item will complete successfully.
  • Flag: The associated [Batch Folder] or [Batch Page] will be flagged.
  • Log: The issue will be logged to the Grooper log.
  • Stop: The [Batch] will be set to error.
Maximum Consecutive Errors
Type: [Int32], Default: 0
The maximum number of consecutive errors, after which a critical stop will be raised. A critical stop will cause services to stop running.
