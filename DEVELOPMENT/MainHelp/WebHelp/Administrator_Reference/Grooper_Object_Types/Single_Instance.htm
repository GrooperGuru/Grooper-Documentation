Grooper 2.70 Documentation

Single Instance

Returns the first instance returned by any extractor and discards all other results. Useful for implementing a waterfall approach where the most strict format is tried first, follow by less strict formats.

Inherits from: [Collation Provider]

Properties

Property Name Description
General
Instance Ranking
Type: [RankingMethod], Default: Order
Controls the order of the result set when selecting a single result. Can be one of the following values:
  • Order: Results are returned in the order in which they appear in the content flow.
  • ReverseOrder: Results are returned in reverse of the order in which they appear in the content flow.
  • Frequency: Results are returned ordered by the number of occurrences of each value in the document. Value which occur more frequently will be sorted to the top.  For example, a mortgage file may contain many different SSN values, but the most frequently occurring SSN value will be that of the borrower.  In such a case, a Pattern-based extraction with a Instance Ranking based on Frequency will produce the correct result for the Borrower SSN field in most cases.
  • Confidence: DO NOT USE.  This ranking method is used internally by Grooper to rank instances extracted by a Field Class during context-based extraction, after the confidence value for each instance has been calculated.
  • OcrConfidence: Results are returned order by the average OCR confidence of all characters which make up the extracted value.
  • ExtractionOrder: Results are returned in the order of the extractor which produced each match. Can be used to prioritize the results from one extractor those of over another.
  • ValueLengthAscending: Results are returned in the order of their value lengths, in ascending order.
  • ValueLengthDescending: Returns are returned in the order of their value lengths, in descending order.
