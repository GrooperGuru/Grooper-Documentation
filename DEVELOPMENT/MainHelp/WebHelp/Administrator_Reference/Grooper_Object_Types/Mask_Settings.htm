Grooper 2.70 Documentation

Mask Settings

Input masking provides detailed control over what users can enter at each character position in a field. Use input masking to ensure correct formatting of manually entered values such as SSNs, phone numbers, and etc.

Inherits from: Grooper Object

Properties

Property Name Description
General
Input Mask
Type: String
An input mask which controls the format in which data must be entered for the field. When enabled, a MaskedTextBox control is used in place of a standard TextBox control. Cannot be used in tables or on fields which are drop-down lists. For a detailed discussion of the special characters using in input masks, see the Microsoft documentation on mask formats.
Reset On Space
Type: Boolean, Default: False
If True, the space input character causes the current editable position in the mask to be reset. If False the Space character will be processed as a normal input character.
Skip Literals
Type: Boolean, Default: False
If True, literal characters in the input masked will be skipped so that the user is not required to enter them.
Hide Prompt On Leave
Type: Boolean, Default: False
If true, the input mask will only be displayed when the field has focus.
Allow Prompt As Input
Type: Boolean, Default: False
Determines whether the prompt character is allowed as valid input.
Text Mask Format
Type: MaskFormat, Default: ExcludePromptAndLiterals
Determines how the literal and prompt characters in the mask are processed when the generating the formatted string. More specifically, it determines whether literal characters, prompt characters, or both are included in the Text property. When prompt characters are excluded, they are transformed into spaces in the formatted string. Can be one of the following values:
  • ExcludePromptAndLiterals: Return only text input by the user.
  • IncludePrompt: Return text input by the user as well as any instances of the prompt character.
  • IncludeLiterals: Return text input by the user as well as any literal characters defined in the mask.
  • IncludePromptAndLiterals: Return text input by the user as well as any literal characters defined in the mask and any instances of the prompt character.
Prompt Character
Type: String, Default: _
The character to be used to indicate the absence of user input.
