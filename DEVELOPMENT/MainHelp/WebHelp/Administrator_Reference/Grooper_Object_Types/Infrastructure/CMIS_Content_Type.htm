CMIS Content Type
A CMIS Content Type represents a content type defined in a [CMIS Repository]. These objects are not created directly, but created when content types are imported from the property panel of a CMIS Repository object. CMIS Content Types are used to create and maintain mappings between Grooper content types and content types in a repository.

Inherits from: [Grooper Node]

Properties

Property Name Description
Export
Export Enabled
Type: [Boolean], Default: False
Determines whether export is enabled for this CMIS Content Type.
Export Content Type
Type: [Content Type]
The Grooper Content Type to be mapped to this CMIS Content Type. (See [Content Type])
Parent Content Types
A list of [Content Types] which may appear on parent folders of the folders being exported.. Allows field values stored on parent folders to be mapped to output values. For example. consider a batch containing legal documents where level 1 folders are classified as Case File, and level 2 folders are classified as Pleading. Case File defines fields Case No, Plaintiff, and Defendant. Pleading defines fields Title and Filed Date. To output all 5 fields to flat database row, run this activity at level 2, specify Pleading as the Content Type, and add Case File to the list of parent content types.
Export Field Mappings
Specifies mappings between CMIS properties and Grooper data model fields.
Export Folder Levels
Type: [Int32], Default: 0
The number of folder levels to be mapped during export.
Export Folder Mappings
When Export Folder Levels is greater than 0, specifies mappings between Grooper Data Model fields and each folder level.
Default Base Folder
The default base CMIS output folder for documents of this type. This value is used for exports when no folder location is specified in the export configuration.
Export Content
Content Format
Type: [Folder Content Generator], Default: Native
Indicates the format in which content files should be exported.
Import
Import Enabled
Type: [Boolean], Default: False
Determines whether import is enabled for this CMIS Content Type.
Import Content Type
Type: [Content Type]
The Grooper Content Type to be used for imported items. (See [Content Type])
Import Field Mappings
Mappings between CMIS element properties and Grooper Data Model properties.
Import Folder Levels
Type: [Int32], Default: 0
Specifies the number of folder levels to create on import.
Import Folder Mappings
When Import Folder Levels is greater than 0, specifies mappings of data elements to each folder level.
Document Type Property
Type: [String]
A CMIS property which will contain the name of a Grooper content type on import. If this property is blank, the Grooper content type assigned will be the Import Content Type. If this property is set, the Grooper content type to be assigned will be a child of the Import Content Type, having the name as the value of the property.
File Name Property
Type: [String]
A CMIS property which will contain filename to be used in Grooper on import. If this property is blank, the Grooper filename will be set to the filename returned by the CMIS repository.
