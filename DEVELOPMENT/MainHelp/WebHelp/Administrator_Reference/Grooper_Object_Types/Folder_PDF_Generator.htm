Grooper 2.70 Documentation

Folder PDF Generator

Defines settings which control the creation of a PDF file from the content associated with a Batch Folder.

Inherits from: Batch Object Stream Generator

Properties

Property Name Description
General
Make Searchable
Type: Boolean, Default: False
If enabled, PDF pages generated from Grooper page images will be generated as searchable PDF pages. Pages must contain OCR data in order for this option to have any effect.
PDF Page Source
Type: PdfPageSourceEnum, Default: Auto
Defines how PDF pages are generated when a page has both an image version and a PDF version. Can be one of the following values:
  • Auto: If the original page was an image-only PDF page, the image version will be used. Otherwise, the PDF version will be used.
  • Image: The image version will always be used.
  • PDF: The PDF version will always be used, if one exists. Otherwise, the image version will be output.
Prefer Child Versions
Type: Boolean, Default: False
If true, the output file generation process will prefer content stored on child objects over content stored on parent objects. By default, if a folder contains a PDF or image-based version, then the folder's content will be included in the output, and any content stored on child objects will not be included. Enabling this option allows the output file to be instead constructed using the content stored on child folder or page objects. For example, in a case where PDF files are imported, and image processing has been applied to the pages, it may be desirable to output the processed page versions rather than the original version stored on the folder.
Page Skip Count
Type: Int32, Default: 0
Indicates the number of pages to be skipped during generation of the output file. Defines the number of pages, starting with the first page, which will be excluded from the output. For example, this option could be used to remove cover sheets from the generated output.
