Header Detector

The header detection process attempts to detect a table header on each page, by analyzing results from the header extractors defined on individual Data Column objects.

A header is defined as any case where a text line (or a sequence of up to N text lines) contains a minimum percentage of header extractor hits. For example, if 3 columns have header extractors, and the Minimum Confidence is 50%, then at least 2 of the header extractors must have matches.

Successful header detection provides significant benefits during table row extraction:

  • Establishes an absolute upper boundary for the first row on each page, in cases where multi-line rows are being extracted.
  • Filters out all header extractor hits which do not appear inside the detected header.
  • Filters out any row candidates appearing above or overlapping with the header.
  • Filters out any footer candidates appearing above or overlapping with the header.
