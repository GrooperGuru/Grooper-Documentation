Normalize
Normalizes character data in the lexicon to the character set of the configured language. This command can be used to normalize imported lexicon data where words are spelled using characters from another language. For example, consider the Russian word 'ничего', which contains two Cyrillic characters ('е' and 'о') which are homoglyphs of the Latin characters 'e' and 'o'. While these characters look exactly the same, they are not the same Unicode characters. A lexicon entry spelled with Latin characters will not match a word extracted from a Russian document which is spelled with Cyrillic characters.

This command will scan all entries in the lexicon. If an extry contains characters which are invalid for the language, then a conversion will be attempted where each invalid character is converted to a corresponding homoglyph which is valid. If this conversion fails, the the entry is considered invalid.

If the conversion is successful, a lookup is performed to locate the controlling entry in the lexicon (i.e. the properly-spelled version). If the lookup fails, then the entry is considered an invalid entry. Otherwise, the controlling entry is updated with any frequency data from the invalid entry, and the invalid entry is discarded.



Inherits from: ObjectCommand(Of Lexicon)

Properties

Property Name Description
General
Save Invalid Entries
Type: Boolean, Default: True
If true, entries which cannot be normalized will be saved to a new lexicon. Otherwise, invalid entries will be deleted. The new lexicon will be created in the same location as the original lexicon, with an " Invalid" suffix.
