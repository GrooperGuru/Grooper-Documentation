Grooper 2.70 Documentation

Text Preprocessor

Defines options for temporary text preprocessing which occurs prior to executing an extractor.

Inherits from: Embedded Object

Properties

Property Name Description
Preprocessing Options
Tab Marking
Type: Horizontal Tab Marker
When enabled, converts space characters to tab characters based on the width of the gap occupied by the space character. Tabs can be matched using \t in regular expressions. Tab marking is useful for identifying the boundaries between segments of text within a line. For example, tabs could be used to break up the columns of a table row based on the fact that there is a gap between the columns. (See Horizontal Tab Marker)
Paragraph Marking
Type: Paragraph Marker
Modifies the input so that CR/LF pairs only appear at the end of paragraphs. By default, OCR results contain a CR/LF pair at the end of each line. Enabling this option will perform paragraph detection and modify the input as follows:
  • If the line represents the last line of a paragraph, the CR/LF pair will be left intact.
  • Otherwise, the CR/LF pair will be replaced by a SPACE character.
Paragraph marking allows extraction of data elements which may wrap from line to line. (See Paragraph Marker)
Vertical Tab Marking
Type: Vertical Tab Marker
When enabled, converts \r\n pairs to \v pairs based on configurable settings. Vertical tab marking is useful for identifying the boundaries between elements in cases where a measurable gap exists. (See Vertical Tab Marker)
Ignore Control Characters
Type: ControlCharacters, Default: None
Indicates the set of control characters to ignore for the purposes of data extraction. A combination of the following flags:
  • None: No characters will be ignored.
  • Space: Space characters will be ignored.
  • NewLine: Carriage return / line feed pairs will be replaced by a single space character.
  • FormFeed: Form feed (\f) characters will be ignored.
  • All: All control characters will be ignored.
Before the expression is executed, all selected characters will be removed from the source document. This option is useful in eliminating data extraction problems caused when OCR engines recognize additional spaces, or for performing extraction across multiple lines.  For example, consider a date value where instead of '03/25/2013', the OCR engine outputs '0 3/25/2 013'.  Ignoring the space character will allow the value to be extracted as a date.
