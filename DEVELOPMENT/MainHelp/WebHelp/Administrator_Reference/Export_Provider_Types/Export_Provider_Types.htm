Grooper 2.70 Documentation

Export Provider

Export providers define export connectivity to external systems such as file systems, content management repositories, mail servers, and etc. Export providers are used by the Export Document activity.

List of Export Provider Types

Name Description
  CMIS Export Provides export connectivity to CMIS-compliant content management platforms. CMIS Export relies on settings defined by CMIS Content Type objects. As such, it is important (a) that any folder to be exported has a Content Type assigned; and (b) that the content type is properly mapped to the desired CMIS Content Type.
  • There must be at least one CMIS Content Type which references the folder type as its 'Export Content Type'.
  • If more than one mapping is present, use the CMIS Repository to specify which mapping should be used.
  • All required fields of the CMIS Content Type must be mapped.
It is important to note that Content Model inheritance applies here. This means that if the folder type is not mapped directly, then any mappings defined at parent levels of the content model will be applied.
  File System Export Provides export connectivity to the Windows file system.
  FTP Export Provides export connectivity over the Ftp and Ftps protocols.
  Mail Export Provides export connectivity to IMAP mail servers. Only documents with a MIME type of message/rfc822 may be exported using this provider.
  SFTP Export Provides export connectivity over the SFTP protocol.
