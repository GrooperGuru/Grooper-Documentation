CMIS Export
Provides export connectivity to CMIS-compliant content management platforms. CMIS Export relies on settings defined by CMIS Content Type objects. As such, it is important (a) that any folder to be exported has a Content Type assigned; and (b) that the content type is properly mapped to the desired CMIS Content Type.
  • There must be at least one CMIS Content Type which references the folder type as its 'Export Content Type'.
  • If more than one mapping is present, use the CMIS Repository to specify which mapping should be used.
  • All required fields of the CMIS Content Type must be mapped.
It is important to note that Content Model inheritance applies here. This means that if the folder type is not mapped directly, then any mappings defined at parent levels of the content model will be applied.

Inherits from: Export Provider

Properties

Property Name Description
General
CMIS Repository
Type: CMIS Repository
The CMIS Repository from which content will be imported. (See CMIS Repository)
Target Folder
The target folder to which items should be written within the CMIS Repository. If no target folder is specified, the Default Base Folder property of the CMIS Content Type will be used. If neither is specified, then an error will be generated.
Update Link
Type: Boolean, Default: True
If true, the Content Link on the document will be updated to point at the exported content.
