Grooper 2.70 Documentation

Extract Features
Divides an image into an NxN matrix of cells and generates a vector of N*N features. The resulting vector of features can be used in real time as pages come off the scanner. Because this classification mechanism does not require full text OCR, and is hardware accelerated, page classification can be achieved in less than 100 ms. Accuracy can be increased by increasing the number of cells. Document types such as invoices (which have a standard header but contain variable information like invoice line items) can be reliably classified by restricting the region of interest.

Inherits from: IP Command

Properties

Property Name Description
General
Region of Interest
Type: Rectangle, Default: (0,0):(0,0)
The region of the page to use for feature extraction. Document types such as invoices (which have a standard header but contain variable information like invoice line items) can be reliably classified by restricting the region of interest.
Feature Types
Type: FeatureType, Default: Intensity
Select the feature types to include. A combination of the following flags:
  • Intensity: Generates a feature for each cell based on its intensity.
  • GradientAngle: Generates a feature for each cell based on its gradient angle. Gradient angle features are only useful on grayscale / color photos.
  • COG_Angle: Generates a feature for each cell based on the angle between the center of the cell and the center of gravity for the black pixels in the cell.
  • COG_Magnitude: Generates a feature for each cell based on the distance between the center of the cell and the center of gravity for the black pixels in the cell.
  • Entropy: Generates a feature for each cell based on the distance between the center of the cell and the center of gravity for the black pixels in the cell.
  • ColorSpaceChannel: Generates a feature for each cell based on the average value for a color space channel.
Most structured document types can be classified accurately using features based on Intensity, or Intensity and COG_Angle.
X Axis Cells
Type: Int32, Default: 8, Range: 1 - 64
The number of cells along the X axis.
Y Axis Cells
Type: Int32, Default: 8, Range: 1 - 64
The number of cells along the Y axis.
Color Space
Type: ColorSpaceType, Default: RGB
The color space to use. Can be one of the following values:
  • RGB: RGB Color Space - Channels: Red, Green, Blue.
  • HSV: HSV Color Space - Channels: Hue, Saturation, Value (Intensity).
  • LAB: LAB Color Space - Channels: Luminosity, A Value, B Value.
  • YCbCr: YCbCr Color Space - Channels: Luma, Chroma Blue, Chroma Red.
  • YUV: YUV Color Space - Channels: Y (luma), U (chroma red), V (chroma blue).
  • XYZ: XYZ Color Space - Channels: X, Y, Z.
  • HLS: HLS Color Space - Channels: Hue, Lightness, Saturation.
The input image will be converted to the indicated color space prior to channel extraction.
Channel
Type: ChannelNumber, Default: Channel1
Indicates the channel to be extracted from the specified color space. Can be one of the following values:
  • Channel1
  • Channel2
  • Channel3
Image Filtering
Enable Filtering
Type: Boolean, Default: True
If set to true, the image will be filtered prior to processing, using the settings specified under 'Image Filtering'.
Filter Command
Type: Filter
Specifies the filtering settings when Image Filtering is enabled. the most common filter to use with this command is a Min filter with a mask size of around 0.1" x 0.1". The objective of filtering is to generate a rough outline of the major elements on the document. (See Filter)
