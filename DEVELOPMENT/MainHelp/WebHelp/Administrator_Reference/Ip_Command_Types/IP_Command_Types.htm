Grooper 2.70 Documentation

IP Command

IP commands perform image processing work, such as image cleanup, format conversion, or feature detection. IP Commands are used to construct IP Profiles, and are also used as stand-alone properties on some Grooper types.

Some IP Commands modify the images they process, while other IP Commands simply detect features in an image, without modifying the image in any way.

Each IP Command defines a set of supported pixel formats. If an image with an unsupported pixel format is encountered, the command will take no action.

Color Adjustment
  Adjust Saturation Adjusts the saturation (color intensity) in an image.
  Adjust Tint Adjusts the tint in an image.
  Auto Adjust Levels The Adjust Levels command corrects the tonal range and color balance by adjusting the shadow, midtone, and highlights of an image.
  Auto Color Balance Automatically corrects the color and contrast of an image.
  Auto White Balance Automatically adjusts the white point in an image.
  Brightness Contrast Modifies the brightness and/or contrast of an image.

  • Replace all colors. - Leave the Target Color empty.
  • Replace all shades of gray. - Leave the Target Color empty, and set Invert to True.
  • Replace a specific color - Set the Target Color to the color which should be replaced.
  • Preserve a specific color - Set the Target Color to the color which should be preserved, and set Invert to True.

  Posterize Generates an image made up of primary and secondary colors of the source image.
  Solarize Use this command to adjust the curves to generate a classic "solarize" effect.
  Swap Channels Converts RGB to BGR.
Feature Detection
  Barcode Detection Detects and reads barcode data.
  Blank Page Detection Determines if a page is blank based on a threshold of black pixels.
  Color Detection Detects whether an image contains color, grayscale, or black and white content, optionally converting the image to the detected color depth.
  Corner Detection Detects corners using Harris corner detection.
  Edge Detection Detects edges in an image using a Sobel filter.
  Line Detection Detects lines in an image using a Hough Transform. This command does not produce an end result by itself, but rather is used in other IP commands, such as Sticky Note Detection, to detect line features in images.
  Patch Code Detection The Detect Patch Codes command is used to determine if an image contains a patch code.  This command is frequently used for Separation.  This command returns true if a patch code is detected and false if one is not.
  Shape Detection Detects regions within an image which are similar to a sample image. This command can be used to detect stamps, seals, logos, and other visual elements in an image. The output from Shape Detection can be used in several ways throughout Grooper:
  • As a trigger for event-based separation in a Separation Profile. If the separation profile is being used in real time during scanning, then a Shape Detection command must be included in the real-time IP Profile being used during Scan.
  • As a source of features for Classification, by including a Shape Detection command in the Feature Extraction IP Profile assigned to a Content Model
  • As a registration mechanism for a Field Zone in zonal extraction. This is useful in cases where a stamp or seal contains information which needs to be extracted. If a Field Zone is configured with a Shape Name, at run time the zone will be positioned directly over the detected shape. Using the IP Profile assigned to the Field Zone, it is then possible to deskew and clean up the region so that OCR is successful.
  Sticky Note Detection Detects color sticky notes, optionally flagging the pages on which they are detected. This command is useful in helping scan operators identify pages with sticky notes attached at scan time, so that any corrective action can be taken before the batch is completed.

This command is designed to detect color sticky notes on non-color document backgrounds. Sticky note detection will not work on documents which have a colored background.

Feature Removal
  Barcode Removal Removes barcodes from an image.
  Blob Removal Removes blobs from the image which meet specific size and fill requirements.

This command is useful for removing objects of a specific size or shape from an image. It can be used to remove small features such as specks, large features such as a company logo, or long, skinny features such as streaks and lines.

It is important when using this command to ensure that the parameters used do not cause unintentional loss of information. For example, when using this command to remove large specks, it is quite possible that legitimate punctuation symbols could also be removed.

General Operation Notes

In image processing terms, a blob is any set of connected pixels in a binary image. As such, the detection portion of this algorithm executes against a binarized version of the input image, created using the settings defined in the Binarization property. When tuning this command, review the "Binarized" output image to ensure the target objects are visible as distinct black blobs. If they are not, try using different binarization settings.

In cases where the input image is already a binary image, the Binarization settings have no effect.

  Halftone Removal Removes halftone regions from an image.

A halftone pattern is a matrix of small dots used to simulate shades of gray on black and white documents, as shown in the example below.

Halftone patterns are commonly used to shade elements such as table headers or section headers, and can prevent OCR engines from recognizing the text inside these regions. Halftone removal is commonly used for temporary pre-OCR image cleanup, to improve recognition of text printed inside halftone regions.

  Hole Punch Removal Removes circular hole punch artifacts from an image. Hole punches produce solid round aritfacts along the edges of an image, as shown below.

NOTES ON INPAINTING: If a hole punch intersects other objects on the image, inpainting can produce undesirable effects by inpainting part of the hole punch with black. To prevent this, the Hole Punch Removal command evaulates each hole punch independently. If the distance from a hole punch to the closest object is less than the Inpaint Radius, the fill method will automatically switch to AutoFill.

See the Wikipedia article on hole punches for information on standard hole punch sizes.

  Line Removal Removes horizontal and vertical lines from an image.
  Negative Region Removal Inverts or removes regions which are inverted (white on black). An negative region is a rectangular area where the feature pixels are white and the background pixels are black, similar to the "Negated" label on the original shown below.

The output from this command is an image where the detected regions are inverted, as shown in the middle, or removed, as shown at the right.

Notes on Inversion

The edges of inverted text regions on scanned images are imperfect, and do not represent straight lines. Because of this, inverting a rectangular region will inevitably leave artifacts around the edge. In most cases, it is best to shrink the detected zone slightly, so that inversion leaves a solid rectangular outline behind. The outline can then be removed cleanly using the Line Dropout command.

Zone shrinking is accomplished by setting the Region Detection / Zone Adjustment property to a negative value.
  Photo Removal Automatically detects and removes photographs.
  Speck Removal Removes small specks from an image.
  Streak Removal Removes vertical streaks from an image.
Format Conversion
  Auto Convert Changes the color depth of an image to match the content of the image. Outputs either a color, grayscale, or black and white based on analysis of the image content:
  • If the image contains color, the output image will be 24-bit color.
  • If the image contains grayscale, the output image will be 8-bit grayscale.
  • Otherwise, the image will be output as black and white.
  Convert Changes the color format of an image.
  Threshold Converts an image to black and white using a variety of thresholding methods.
Image Analysis
  Auto QA Automatically flags images which meet certain criteria, such as being too dark, too light, or too busy.
  Compute Moments Calculates Hu Moments for an image. Calculates the 7 invariant Image Moments described in M. K. Hu, "Visual Pattern Recognition by Moment Invariants", IRE Trans. Info. Theory, vol. IT-8, pp.179–187, 1962
  Extract Features Divides an image into an NxN matrix of cells and generates a vector of N*N features. The resulting vector of features can be used in real time as pages come off the scanner. Because this classification mechanism does not require full text OCR, and is hardware accelerated, page classification can be achieved in less than 100 ms. Accuracy can be increased by increasing the number of cells. Document types such as invoices (which have a standard header but contain variable information like invoice line items) can be reliably classified by restricting the region of interest.
  Histogram Calculates an image histogram.
  Measure Entropy Measures the entropy (amount of change) in an image.
  Projection Profile Generates a projection profile histogram.
Image Transforms
  Auto Deskew Detects the skew angle of an image, and optionally rotates the image to compensate. This command works by rotating a low-resolution copy of the image through a sequence of angles, and analyzing the image at each orientation to determine how straight it is.

The angles to be tested are defined by Maximum Angle and Angle Precision. The algorithm will iterate from -Maximum Angle to +Maximum Angle in increments of Angle Precision, testing the straightness of the image at each angle. For example, a 5° maximum angle with 0.25° angle precision will test 41 different angles: [-5.00°, -4.75°, -4.50°, ..., 0.00°, ..., 4.50°, 4.75°, 5.00°]. Therefore, a higher maximum angle or lower angle precision will result in more distinct angles being tested, reducing the speed of the deskew operation, but increasing its angular detection range or accuracy.

  Auto Deskew Basic Automatically deskews a document image. Speed can be improved by reducing the Angle Limit, reducing the Precision, or by limiting the Area of Interest to a smaller rectangular area containing the horizontally aligned text area. The Coverage property can also be decreased to increase speed; however, doing so could result in loss of accuracy. For guaranteed accuracy, set the Coverage property to 100.
  Auto Orient Determines the correct orientation for an image and rotates the image by analyzing the orientation of the characters on the page.
  Resize Resizes an image.
  Rotate Rotates an image by a given angle.
