Auto Deskew Basic
Automatically deskews a document image. Speed can be improved by reducing the Angle Limit, reducing the Precision, or by limiting the Area of Interest to a smaller rectangular area containing the horizontally aligned text area. The Coverage property can also be decreased to increase speed; however, doing so could result in loss of accuracy. For guaranteed accuracy, set the Coverage property to 100.

Inherits from: IP Command

Properties

Property Name Description
General
Area of Interest (inches)
Type: Rectangle, Default: (0,0):(0,0)
The rectangular area that will be used for determining the skew angle (specified in inches)
Background Color
Type: Color, Default: White
The background color to use for filling areas left empty by the deskew process. The background color is only relevant when the Crop Mode is set to None or OriginalSize.
Coverage %
Type: Double, Default: 8, Range: 0 - 100
Represents the percentage of the image that will be randomly sampled when determining the skew angle. Given the random nature of the coverage, low values result in much faster performance than 100% coverage with generally the same results. A default value of 8 was derived after tests with many images. This results in accurate skew angle detection with fast speed. This property is exposed for edge cases where 8 is not sufficient. When accuracy is of utmost importance at the expense of speed, set this value to 100.
Crop Mode
Type: DeskewCropMode, Default: FullCrop
A value indicating the method of cropping the resulting de-skewed image. Can be one of the following values:
  • None: Do not crop the deskewed image. The resulting image will be larger than the source image.
  • FullCrop: Crop all of the excess rotated area. The resulting image will be smaller than the source image.
  • OriginalSize: Crop the deskewed image so that it is exactly the same size as the original image.
Extract Boundary
Type: Boolean, Default: True
Enables or disables the Extract Boundary property. If enabled, will pre-process the image by removing solid filled areas. Disabling this option will improve performance at the expense of accuracy. If the target images are known not to have solid filled areas, then this option may be safely disabled.
Angle Limit
Type: Double, Default: 5°, Range: 0° - 90°
The maximum skew angle that will be detected. Decreasing this value improves performance, while increasing improves the chance that a drastically skewed image will be straightened.
Precision
Type: Int32, Default: 4, Range: 1 - 64
This value is the inverse of the smallest angle increment that the algorithm will find For example:

1 = 1 degree precision
2 = 1/2 degree precision
4 = 1/4 degree precision, etc.

The smaller the value the more time it takes to calculate the de-skew angle.
