Shape Detection
Detects regions within an image which are similar to a sample image. This command can be used to detect stamps, seals, logos, and other visual elements in an image. The output from Shape Detection can be used in several ways throughout Grooper:
  • As a trigger for event-based separation in a Separation Profile. If the separation profile is being used in real time during scanning, then a Shape Detection command must be included in the real-time IP Profile being used during Scan.
  • As a source of features for Classification, by including a Shape Detection command in the Feature Extraction IP Profile assigned to a Content Model
  • As a registration mechanism for a Field Zone in zonal extraction. This is useful in cases where a stamp or seal contains information which needs to be extracted. If a Field Zone is configured with a Shape Name, at run time the zone will be positioned directly over the detected shape. Using the IP Profile assigned to the Field Zone, it is then possible to deskew and clean up the region so that OCR is successful.


Inherits from: IP Command

Properties

Property Name Description
General
Sample Images
Type: Sample Image Collection
One or more sample images representing the shape to be detected. (See Sample Image Collection)
Shape Name
Type: String
The name to the used to identify the shape. If no name is provided, the name of the sample image which matched each result will be used.
Proximity Measure
Type: Method, Default: SAD
Determines the method used for calculating image proximity. Can be one of the following values:
  • SAD: Sum of Absolute Differences.
  • SSD: Sum of Squared Distances.
  • CrossCorr: Normalized cross-correlation.
Minimum Confidence
Type: Double, Default: 80%, Range: 0% - 100%
The minimum confidence for a match to be considered successful.
Maximum Angle
Type: Double, Default: 20°
The maximum angle at which shapes will be detected. This property, in combination with the Angle Step, determine the number of discreet angles which will be tested.
Angle Step
Type: Double, Default: 1°
The angular increment. For example, a Maximum Angle of 25° and an Angle Step of 5° will test the following angles: -25°, -20°, -15°, -10°, -5°, 0°, 5°, 10°, 15°, 20°, 25°. The Maximum Angle must be an even multiple of the Angle Step.
Preprocessing
Processing Resolution
Type: ProcessingResolution, Default: Dpi25
The resolution at which this command will be processed. Can be one of the following values:
  • FullResolution
  • Dpi10
  • Dpi15
  • Dpi20
  • Dpi25
  • Dpi30
  • Dpi40
  • Dpi50
  • Dpi75
  • Dpi100
  • Dpi150
  • Dpi200
  • Dpi250
  • Dpi300
Binarization
Type: Binarize
If enabled, defines settings which binarize the image prior to processing. (See Binarize)
Dilation Factor
Type: Int32, Default: 4
The amount of dilation to apply to images before processing.
Region of Interest (inches)
Type: Rectangle, Default: (0,0):(0,0)
The rectangular region to be search for the Shape.
Command Info
Command Type
Type: String
The type of IP command.
Supported Pixel Formats
Type: String
The pixel formats supported by this command.
