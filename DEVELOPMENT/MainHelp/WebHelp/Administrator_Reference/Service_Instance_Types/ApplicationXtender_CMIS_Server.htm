ApplicationXtender CMIS Server

Exposes an EMC ApplicationXtender repository over CMIS. For a specific AX data source, exposes each of the AX applications as a CMIS repository. Each repository has a single content type with properties matching the field layout of the corresponding AX application.

Inherits from: Web Service

Properties

Property Name Description
AppXtender Connection
AX Web Services URL
Type: String
The URL for an instance of ApplicationXtender Web Services.
AX Data Source Name
Type: String
The ApplicationXtender Data Source name to use for this CMIS Service.
Use Windows Credentials
Type: Boolean, Default: False
If enabled, the service will connect to AX Web Services using the Windows credentials under which the service is running. Otherwise, an anonymous connection will be created. If Windows security is used to control access to the AX Web Services URL, this option must be enabled.
Connection Time Out
Type: Int32, Default: 0
The timeout, in seconds, for calls from this service instance to AppXtender Web Services.
Load Timeout
Type: Int32, Default: 0
The timeout, in seconds, for generating the content stream for an individual document. This value represents the total amount of time a document render or a direct page export operation may take before a timeout error is generated. If large documents are being extracted from the AppXtender repository, it may be necessary to increase this value.
Document Rendering
Rendering Mode
Type: RenderingMode, Default: ImageOnly
Controls how document content is extracted from ApplicationXtender. Can be one of the following values:
  • ImageOnly: All documents will be returned as multipage TIFF images. In this mode, all documents will be reported to have a MIME type of image/tiff. In the case where all pages of a document are image-based types, the document's content stream will be a multi-page TIFF containing all pages of the document. PDF documents and text files will be automatically converted to images. For all others types of pages, such as AX foreign files, a placeholder image will be generated to represent the page.
  • Native: Documents will be returned in their native format. In this mode, the MIME type for each document is determined examining individual pages. Use this mode when it is necessary interchange electronic documents between Grooper and ApplicationXtender. Please note that when Native mode is enabled, determining the MIME type for a document can be a time-consuming operation which slows down query operations. This is due to the fact that in AX, the MIME type associated with a document can only be determined by reading the header of each content file - an operation which adds time to queries requesting cmis:contentStreamMimeType, cmis:contentStreamFileName, or cmis:contentStreamLength.
  • RenderServer: AX Render Server will be used to render documents to TIFF or PDF, based on the value of the 'Render Format' property. Use this mode when it is necessary to include AX annotations on the images ingested to Grooper, or to render AX COLD documents.
Render Format
Type: FileType, Default: PDF
Indicates the file format to be requested from AX Render Server. Can be one of the following values:
  • PDF: Slower performance, but can convert foreign file formats.
  • TIF: Faster performance, but cannot convert foreign file formats.
Render Annotations
Type: Boolean, Default: False
If enabled, ApplicationXtender annotations will be rendered.
Form Overlay Mode
Type: FormOverlayType, Default: None
The form overlay mode to be used when rendering COLD documents. Can be one of the following values:
  • None: No form overlay.
  • Image: Image-based form overlay.
  • Text: Text-based form overlay.
Generate Error Page Image
Type: Boolean, Default: False
If enabled, an error page will be generated when an AX document page cannot be retrieved from the AX Web Service.
Document Save Options
Merge Action
Type: MergeAction, Default: None
Controls whether documents are added as new documents, or merged with existing documents based on matching index values. Can be one of the following values:
  • None: All documents will be added as new ApplicationXtender documents.
  • Append: Adds the current document's pages to the end of the existing document in ApplicationXtender.
  • Prepend: Inserts the current document's pages to the beginning of the existing document in ApplicationXtender.
This feature requires that the target AX Application have the 'Part of Unique Key' flag enabled on at least one index field. Fields with this flag enabled will be used in a query prior to each document being exported. If a document is returned, the export will append/pre-pend the current Grooper document. Otherwise, a new document will be exported.
Submit Full Text Job
Type: Boolean, Default: False
New or modified documents will be submitted to the full text index server configured for ApplicationXtender. This option requires the user account configured for the service's login, have administrative privileges on the repository server. An exception will be thrown if this method is called by an account without administrative privileges.
Full Text Job Queue Name
Type: String
The name of the ApplicationXtender Full Text Job Queue to which documents should be submitted.
Direct Database Connection
Enable Direct Database Connection
Type: Boolean, Default: False
Enable a direct connection to the ApplicationXtender MS SQL database. A direct database connection will improve performance when performing CMIS queries against a repository, and during Grooper Batch importing.
Data Connection
Type: Data Connection
A Grooper Data Connection to the ApplicationXtender MS SQL database. Grooper Data Connections are created in Grooper Administration within the Infrastructure Folder. (See Data Connection)
Web Service Settings
Connection URL
Type: String
Convenience string property containing the full URL path for this web service. The URL path is constructed by combining the Host Name, Port Number and URL Path property values and observing the HTTPS Enabled property. For example, this property's string value can easily be copied to the clipboard, and pasted into the URL property for a CMIS Connection.
HTTPS Enabled
Type: Boolean, Default: False
Controls whether the HTTPS protocol is enabled.
Host Name
Type: String
The host name to be used for this web service instance.
Port Number
Type: Int32, Default: 0
The port number to be used for the web service.
URL Path
Type: String
The web server path on which the CMIS server should be hosted.
Enable Debugging
Type: Boolean, Default: False
Controls whether exception detail is returned by this web service when an exception occurs.
Service Settings
Repository
Type: Repository Connection
The repository to which this service instance in bound. Service instances are bound permanently to a Grooper repository. To change to repository, delete the service instance and reinstall it using the desired repository.
Service Display Name
Type: String
The display name for the service.
Thread Priority
Type: ThreadPriority, Default: Lowest
The priority level with which threads will be created. Can be one of the following values:
  • Lowest: The can be scheduled after threads with any other priority.
  • BelowNormal: The can be scheduled after threads with Normal priority and before those with Lowest priority.
  • Normal: The can be scheduled after threads with AboveNormal priority and before those with BelowNormal priority. Threads have Normal priority by default.
  • AboveNormal: The can be scheduled after threads with Highest priority and before those with Normal priority.
  • Highest: The can be scheduled before threads with any other priority.
If the service is running on a workstation, a priority lower than normal will help prevent the service from impacting the user interface responsiveness.
User Name
Type: String
The user account under which the service should run. The account specified must:
  • Have the Logon as Service permission on the machine on which the service is to run.
  • Have access to the Grooper database, if Windows security is being used for SQL connections.
  • Have access to the Grooper file store location.
Password
Type: String
The password for the user account under which the service should run.
