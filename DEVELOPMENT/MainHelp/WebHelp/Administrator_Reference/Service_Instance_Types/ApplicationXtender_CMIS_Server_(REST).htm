ApplicationXtender CMIS Server (REST)

Exposes an EMC ApplicationXtender repository over CMIS. For a specific AX data source, exposes each of the AX applications as a CMIS repository. Each repository has a single content type with properties matching the field layout of the corresponding AX application.

Properties

Property Name Description
AppXtender Connection
AX Web Services URL The URL for an instance of ApplicationXtender Web Services.
Use Windows Credentials If enabled, the service will connect to AX Web Services using the Windows credentials under which the service is running. Otherwise, an anonymous connection will be created. If Windows security is used to control access to the AX Web Services URL, this option must be enabled.
Connection Time Out The timeout, in seconds, for calls from this service instance to AppXtender Web Services.
Load Time Out The timeout, in seconds, for generating the content stream for an individual document. This value represents the total amount of time a document render or a direct page export operation may take before a timeout error is generated. If large documents are being extracted from the AppXtender repository, it may be necessary to increase this value.
Document Rendering
Use Render Server Controls when ApplicationXtender Render Server is used for document rendering. Can be one of the following values:
  • Automatic: Render Server will be used selectively, under the following circumstances:
    • If any pages of the document are foreign files.
    • If Render Annotations is enabled, and the document has annotations.
    In all other cases,
  • Enabled: Render Server will be used for every document.
  • Disabled: Render Server will never be used.
This CMIS server supports 2 primary mechanisms for obtaining document content:
  • Direct Export - Each page of the document is extracted from AX Web Services as an image, and the sequence is streamed back as a multi-page TIFF file. For pages referencing foreign files, a placeholder image will be inserted. This is the fastest method of extracting content, and is the best choice for image-only repositories.
  • Render Server - ApplicationXtender Render Server is used to render all pages of the document into a multipage TIFF of PDF file. This method requires more processing time than direct export, but has the advantage of allowing foreign files and annotations to be rendered.
File Format Indicates the file format to be requested from Render Server. Can be one of the following values:
  • PDF: Faster performance, but cannot convert foreign file formats.
  • TIF: Slower performance, but can convert foreign file formats.
Render Annotations If enabled, ApplicationXtender annotations will be rendered.
Form Overlay Mode The form overlay mode to be used when rendering COLD documents. Can be one of the following values:
  • Text: Text-based form overlay.
  • Image: Image-based form overlay.
  • None: No form overlay.
Document Save Options
Merge Action Controls whether documents are added as new documents, or merged with existing documents based on matching index values. Can be one of the following values:
  • None: All documents will be added as new ApplicationXtender documents.
  • Append: Adds the current document's pages to the end of the existing document in ApplicationXtender.
  • Prepend: Inserts the current document's pages to the beginning of the existing document in ApplicationXtender.
This feature requires that the target AX Application have the 'Part of Unique Key' flag enabled on at least one index field. Fields with this flag enabled will be used in a query prior to each document being exported. If a document is returned, the export will append/pre-pend the current Grooper document. Otherwise, a new document will be exported.
Submit Full Text Job New or modified documents will be submitted to the full text index server configured for ApplicationXtender. This option requires the user account configured for the service's login, have administrative privileges on the repository server. An exception will be thrown if this method is called by an account without administrative privileges.
Full Text Job Queue Name The name of the ApplicationXtender Full Text Job Queue to which documents should be submitted.
Web Service Settings
Connection URL Convenience string property containing the full URL path for this web service. The URL path is constructed by combining the Host Name, Port Number and URL Path property values and observing the HTTPS Enabled property. For example, this property's string value can easily be copied to the clipboard, and pasted into the URL property for a CMIS Connection.
HTTPS Enabled Controls whether the HTTPS protocol is enabled.
Host Name The host name to be used for this web service instance.
Port Number The port number to be used for the web service.
URL Path The web server path on which the CMIS server should be hosted.
Enable Debugging Controls whether exception detail is returned by this web service when an exception occurs.
Service Settings
Repository The repository to which this service instance in bound. Service instances are bound permanently to a Grooper repository. To change to repository, delete the service instance and reinstall it using the desired repository.
Service Display Name The display name for the service.
Thread Priority The priority level with which threads will be created. Can be one of the following values:
  • Lowest: The can be scheduled after threads with any other priority.
  • BelowNormal: The can be scheduled after threads with Normal priority and before those with Lowest priority.
  • Normal: The can be scheduled after threads with AboveNormal priority and before those with BelowNormal priority. Threads have Normal priority by default.
  • AboveNormal: The can be scheduled after threads with Highest priority and before those with Normal priority.
  • Highest: The can be scheduled before threads with any other priority.
If the service is running on a workstation, a priority lower than normal will help prevent the service from impacting the user interface responsiveness.
User Name The user account under which the service should run. The account specified must:
  • Have the Logon as Service permission on the machine on which the service is to run.
  • Have access to the Grooper database, if Windows security is being used for SQL connections.
  • Have access to the Grooper file store location.
Password The password for the user account under which the service should run.
