Section Extract Method

A Section Extract Method defines an extraction method for use with Data Section objects.

List of Section Extract Method Types

Name Description
Divider Identifies Section Instances by splitting the input content at each occurence of a specific text pattern. This method is useful for processing content with repeating sections of information which occur sequentially in the text flow of the document.

For example, the document below could be divided into employee records by splitting the content at each occurence of the text pattern '\r\nEMPLOYEE:'.


ACTIVE EMPLOYEE REPORT

EMPLOYEE: Doe, Jane SSN: 000-00-0000 PHONE: (000) 000-0000
ADDRESS: 1234 S Main CITY: Anytown STATE: AA ZIP: 00000
DOB: 00/00/0000 EMAIL: janedoe@acme.com DEPARTMENT: IT

EMPLOYEE: Doe, John SSN: 000-00-0000 PHONE: (000) 000-0000
ADDRESS: 2233 N Elm CITY: Anytown STATE: AA ZIP: 00000
DOB: 00/00/0000 EMAIL: johndoe@acme.com DEPARTMENT: AP

EMPLOYEE: Smith, Kim SSN: 000-00-0000 PHONE: (000) 000-0000
ADDRESS: 8888 W 24th CITY: Anytown STATE: AA ZIP: 00000
DOB: 00/00/0000 EMAIL: kimsmith@acme.com DEPARTMENT: Admin
Fixed Identifies section instances using a fixed geometric region. Outputs a single section instance containing all character data bounded by a rectangular region on a specific page. If the document does not contain the page specified, an empty result set will be returned.
Flow Identifies Section Instances using an extractor to match segments of text. For each match, the character data between the starting and ending position will become an output Section Instance. This method is useful in cases where (a) the section content is contiguous in the input; and (b) the start and end of the section content can be identified with pattern matching.

For example, the document below could be divided into employee records using the pattern 'EMPLOYEE:.*?DEPARTMENT:[^\r]+'.

Geometric Defines a rectangular region using anchors, and extracts the character data bounded by the region as a Section Instance. Geometric section extraction is performed as follows:
  1. The Main Extractor establishes the initial region.
  2. Each edge of the region can then be adjusted using text anchors, page boundaries, or manual rules.
  3. The result can then be expanded to detected lines which bound the region..
NOTE: This method cannot be used in cases where the section content spans multiple pages.
