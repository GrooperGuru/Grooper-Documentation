Grooper 2.70 Documentation

Collation Provider

Collation providers define various methods of processing the the output of a Data Type object.

List of Collation Provider Types

Name Description
Array Matches a list of values arranged in horizontal, vertical, or flow order. This collation method searches through the results from all child extractors and locates instances where N or more matches are found in an arrangement which matches one of the enabled Array Layout types. For each such match, the child result instances will be combined (see the Combine Method property) and returned as a single result, which contains the source child extractor instances as children.
Combine Combines instances from child extractors based on the grouping specified in the Group By property.
Individual Returns all instances from all extractors individually.
Ordered Array Finds sequences of values where one result is present for each extractor, in the order in which they appear. For example, assume Extractor 1 matches a name, Extractor 2 matches a street address line, and Extractor 3 matches a city/state/zip line. Using the vertical array layout, this could be used to match:

John Doe
13900 N Harvey
Edmond, OK 73013

In horizontal or flow layout, it could match the same information laid out this way:

John Doe, 13900 N Harvey, Edmond, OK 73013

For each such match, the child result instances will be combined (see the Combine Method property) and returned as a single result, which contains the source child extractor instances as children.
Pattern-Based Uses a regular expression to select a sequence of child extractor results. Child extractor results are referenced using @ExtractorName, where ExtractorName is the name of a child (or referenced) extractor. For example, consider the tabular data below, which represents information from a college transcript:

GE140 WORLD CIVILIZATION I 3.00 A 12.00
PSY212 GENERAL PSYCHOLOGY 3.00 A 12.00
GE185 HEALTH CONCEPTS 2.00 C 4.00

Three extractors are created for the Data Type:

  • Course No - Matches 'GE140', 'PSY212', etc.
  • Decimal - Matches '3.00', '12.00', etc.
  • Letter Grade - Matches 'A', 'C', etc.

The following collation expression could then be used to select the entire line:

@Course_No .*? @Decimal @Letter_Grade @Decimal

The expression can be further expanded to include group names, mapping values directly to table column names:

(?<Course_No>@Course_No)
(?<Description>[^\r]*?)
(?<Hours>@Decimal)
(?<Grade>@Letter_Grade)
(?<Points>@Decimal)
Single Instance Returns the first instance returned by any extractor and discards all other results. Useful for implementing a waterfall approach where the most strict format is tried first, follow by less strict formats.
Split Splits the input at each instance found by an extractor. The Split Position property provides a variety of methods for splitting the input.
